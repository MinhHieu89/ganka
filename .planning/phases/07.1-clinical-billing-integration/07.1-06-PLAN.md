    1. Create `useBillingHub.ts` custom hook:
       - Uses `HubConnectionBuilder` from `@microsoft/signalr`
       - Connects to `/api/hubs/billing` with JWT access token from auth store
       - `withAutomaticReconnect()` for automatic reconnection
       - `configureLogging(LogLevel.Warning)` for production-appropriate logging
       - On connect: invoke `JoinCashierDashboard`
       - On reconnect: re-invoke `JoinCashierDashboard` (connection group lost on reconnect)
       - Listen for events:
         - `InvoiceCreated` -> invalidate `["pending-invoices"]` query
         - `LineItemAdded` -> invalidate `["pending-invoices"]` and `["invoice", invoiceId]` queries
         - `InvoiceVoided` -> invalidate `["pending-invoices"]` query
       - Cleanup: `LeaveCashierDashboard` + `stop()` on unmount
       - Use TanStack Query invalidation (simpler than direct state update, leverages existing data fetching)
    2. Integrate into existing CashierDashboard component (or the billing index page):

       - Call `useBillingHub()` at the top of the cashier dashboard page component
       - Add a small connection status indicator (green dot = connected, yellow = reconnecting, red = disconnected) using shadcn/ui Badge
       - The hook handles all SignalR logic -- the component just calls the hook and shows status
    3. Handle token refresh: `accessTokenFactory` should return fresh access token. Read the auth store pattern from existing frontend code to get the current token.
