<objective>
Create the ServiceCatalogItem entity in the Billing module with full CRUD handlers, EF Core configuration, repository, default seeder, API endpoints, and unit tests. This entity stores configurable prices for clinical services (consultation, follow-up, etc.) that billing event handlers will use to auto-price invoice line items.

Purpose: The service catalog is the pricing source for clinical exam fees. When a visit is created, the billing handler looks up the consultation fee from this catalog.
Output: ServiceCatalogItem entity, CRUD handlers, API endpoints, seeder with default services, migration, unit tests.
