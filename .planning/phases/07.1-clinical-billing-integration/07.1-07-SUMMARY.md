## One-liner
End-to-end verification of clinical-billing integration with critical fixes for domain event dispatching and BranchId propagation.

## Key files

### Created
- `backend/src/Shared/Shared.Infrastructure/Interceptors/DomainEventDispatcherInterceptor.cs` — EF Core SaveChanges interceptor that dispatches domain events to Wolverine

### Modified
- All integration events across Clinical, Pharmacy, Optical, Treatment — added BranchId parameter
- All domain events — added BranchId parameter
- All aggregate roots raising events — pass BranchId.Value
- All cascading publish handlers — forward BranchId
- All billing event handlers — use @event.BranchId instead of ICurrentUser
- `backend/src/Shared/Shared.Infrastructure/IoC.cs` — register DomainEventDispatcherInterceptor
- `backend/src/Bootstrapper/Program.cs` — add interceptor to DbContext configuration

## Decisions
- **Domain event dispatching via EF Core interceptor**: Chose SaveChangesInterceptor over modifying UnitOfWork because it's cross-cutting and applies to all modules automatically
- **IServiceProvider injection**: Interceptor is singleton (required by DbContext options lifetime) but IMessageBus is scoped, so we create a scope to resolve it
- **BranchId in events**: Passed through the entire event chain (domain event → integration event → handler) instead of relying on ICurrentUser which is unavailable in async message handler context

## Deviations
1. **[Critical Fix] Domain events never dispatched**: Domain events were collected on AggregateRoot but SaveChangesAsync never published them to Wolverine. Created DomainEventDispatcherInterceptor to fix.
2. **[Critical Fix] BranchId empty in handlers**: All billing handlers used ICurrentUser.BranchId which throws when running outside HTTP context (async Wolverine handlers). Added BranchId to all integration events.
