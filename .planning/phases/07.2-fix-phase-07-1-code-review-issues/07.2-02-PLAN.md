1. **HandleVisitCancelled.cs (CR-03):** Change the guard from:
   `if (invoice is null || invoice.Status == InvoiceStatus.Voided) return;`
   to:
   `if (invoice is null || invoice.Status != InvoiceStatus.Draft) return;`
   This ensures only Draft invoices can be voided on visit cancellation. Finalized (paid) invoices are protected.

2. **HandleDrugDispensed.cs (CR-06):** Add idempotency check before the foreach loop that adds line items:
   ```csharp
   // Idempotency: skip items already billed from this dispensing event
   var existingSourceIds = invoice.LineItems
       .Where(li => li.SourceType == "Dispensing" && li.SourceId == @event.VisitId)
       .Select(li => li.Description)
       .ToHashSet();

   foreach (var item in @event.Items)
   {
       if (existingSourceIds.Contains(item.DrugName)) continue;
       invoice.AddLineItem(...);
   }
   ```

3. **HandleGlassesOrderCreated.cs (CR-06):** Add idempotency check:
   ```csharp
   // Idempotency: skip items already billed from this glasses order
   var existingSourceIds = invoice.LineItems
       .Where(li => li.SourceType == "GlassesOrder" && li.SourceId == @event.OrderId)
       .Select(li => li.Description)
       .ToHashSet();

   foreach (var item in @event.Items)
   {
       if (existingSourceIds.Contains(item.Description)) continue;
       invoice.AddLineItem(...);
   }
   ```

4. **IntegrationEventHandlerTests.cs:** Add new test cases:

   - `HandleDrugDispensed_DuplicateEvent_DoesNotAddDuplicateLineItems`: Create invoice with existing dispensing line items, send same event again, assert line item count unchanged.
   - `HandleGlassesOrderCreated_DuplicateEvent_DoesNotAddDuplicateLineItems`: Same pattern for glasses orders.
