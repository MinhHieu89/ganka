  <action>
1. Change `DomainEventDispatcherInterceptor` to accept `IMessageBus` via constructor instead of `IServiceProvider`:
   - Replace `private readonly IServiceProvider _serviceProvider;` with `private readonly IMessageBus _messageBus;`
   - Update constructor: `public DomainEventDispatcherInterceptor(IMessageBus messageBus)`
   - In `DispatchDomainEventsAsync`: remove `using var scope = _serviceProvider.CreateScope();` and `var messageBus = scope.ServiceProvider.GetRequiredService<IMessageBus>();`
   - Use `_messageBus.PublishAsync(domainEvent)` directly
   - Remove `using Microsoft.Extensions.DependencyInjection;` if no longer needed

2. In `IoC.cs`, change registration from Singleton to Scoped:
   - Replace `services.AddSingleton<DomainEventDispatcherInterceptor>();` with `services.AddScoped<DomainEventDispatcherInterceptor>();`

This works because all DbContexts are registered as Scoped, and `sp.GetRequiredService<DomainEventDispatcherInterceptor>()` in Program.cs resolves from the current scope. The IMessageBus injected will be the same scope as the Wolverine handler pipeline, ensuring domain events participate in the transactional outbox.

1. Create a new EF Core migration that:
   - Creates a SQL SEQUENCE: `CREATE SEQUENCE billing.InvoiceNumberSeq START WITH 1 INCREMENT BY 1;`
   - Adds a unique index on `Invoices.InvoiceNumber`: `CREATE UNIQUE INDEX IX_Invoices_InvoiceNumber ON billing.Invoices (InvoiceNumber);`
   - To determine starting value, check if existing invoices exist and set START WITH accordingly. If the schema already has invoices, use a data fixup: `DECLARE @max INT = (SELECT ISNULL(MAX(CAST(RIGHT(InvoiceNumber, 5) AS INT)), 0) FROM billing.Invoices); IF @max > 0 ALTER SEQUENCE billing.InvoiceNumberSeq RESTART WITH @max + 1;`

2. Rewrite `InvoiceRepository.GetNextInvoiceNumberAsync`:
   ```csharp
   public async Task<string> GetNextInvoiceNumberAsync(int year, CancellationToken ct)
   {
       var nextVal = await context.Database
           .SqlQueryRaw<long>("SELECT NEXT VALUE FOR billing.InvoiceNumberSeq AS Value")
           .FirstAsync(ct);
       return $"HD-{year}-{nextVal:D5}";
   }
   ```

3. Run `dotnet ef migrations add AddInvoiceNumberSequence --project backend/src/Modules/Billing/Billing.Infrastructure --startup-project backend/src/Bootstrapper` and then apply with `dotnet ef database update`.

Note: The SEQUENCE is not year-scoped (it increments globally). This is acceptable since the year prefix in the format string handles the year display, and gaps in the sequence within a year are fine for invoice numbering.
