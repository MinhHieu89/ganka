## Decisions Made
- Changed `optionsLifetime` from `Singleton` to `Scoped` in `ConfigureDbContext` -- required because scoped interceptors cannot be resolved from the root service provider; this ensures each DbContext scope gets the correct interceptor instance
- SQL SEQUENCE is global (not year-scoped) since the year prefix in the format string handles year display, and gaps within a year are acceptable for invoice numbering
- Unique index on InvoiceNumber already existed from InitialBilling migration, so no duplicate creation was needed in the new migration
