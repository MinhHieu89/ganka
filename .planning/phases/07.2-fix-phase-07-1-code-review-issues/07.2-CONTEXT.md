<decisions>
## Implementation Decisions

### Critical Backend Fixes (must fix)

1. **DomainEventDispatcherInterceptor scope issue** — Singleton interceptor creates orphaned scope for IMessageBus, bypassing Wolverine transactional outbox. Fix: use Wolverine's built-in integration or inject IMessageBus from the current scope rather than creating a new one.

2. **Invoice number race condition** — `GetNextInvoiceNumberAsync` in InvoiceRepository has no database-level lock. Concurrent handlers can produce duplicate invoice numbers. Fix: use SQL SEQUENCE or `SELECT ... WITH (UPDLOCK, HOLDLOCK)` + add unique index on InvoiceNumber as safety net.

3. **HandleVisitCancelled can void Finalized invoices** — Only checks for Voided status, should also guard against Finalized (paid). Fix: change guard to `invoice.Status != InvoiceStatus.Draft` so only Draft invoices are voided on visit cancellation.

4. **HandleTreatmentSessionCompleted creates invoices with empty PatientName** — TreatmentSessionCompletedIntegrationEvent doesn't carry PatientName. Fix: add PatientName to the event chain (domain event -> integration event -> handler).

5. **GlassesOrder.VisitId nullability mismatch** — Non-nullable in entity, Guid? in events. Fix: align to non-nullable in events since entity enforces it.

### Important Backend Fixes

6. **No idempotency guard on HandleDrugDispensed / HandleGlassesOrderCreated** — Wolverine at-least-once redelivery can create duplicate line items. Fix: check for existing line items with same SourceId before adding.

7. **HandleOtcSaleCompleted uses Guid.Empty for anonymous patients** — Should use nullable PatientId. Fix: make Invoice.PatientId nullable or use a dedicated IsAnonymous flag.

8. **GetServiceCatalogItemByCode doesn't uppercase** — Create normalizes to uppercase, but lookup doesn't. Fix: add `.ToUpperInvariant()` to the query handler.

9. **No role authorization on service catalog write endpoints** — Any authenticated user can modify prices. Fix: add role-based policy (Admin/Manager) on POST/PUT endpoints.

10. **Missing unit tests for Publish*IntegrationEvent handlers** — 6 domain-to-integration event mapping handlers have no tests. Fix: add unit tests per TDD requirement.

11. **GlassesOrder.DescriptionVi hardcoded empty** — Vietnamese descriptions blank for optical items. Fix: source from catalog or add DescriptionVi to GlassesOrderItem.

12. **Off-catalog dispensing lines produce zero-price billing items** — No guard on zero-price line items. Fix: skip zero-price items or log warning.

### Critical Frontend Fixes

13. **SignalR connection rebuilt on re-renders** — useEffect dependency array includes callbacks that can change. Fix: use useRef pattern for queryClient, stable empty dependency array for connection effect.

14. **mutateAsync without try/catch** — Unhandled promise rejection in ServiceCatalogFormDialog. Fix: wrap in try/catch.

15. **Zod validation messages hardcoded English** — Violates i18n requirement. Fix: use translation function in schema factory.

### Important Frontend Fixes

16. **TanStack Router `as string` cast** — Bypasses typed `to` prop safety. Fix: use typed params API.

17. **Dead UI buttons (Open/Close Shift)** — No onClick handlers. Fix: disable with tooltip or remove until shift management is wired.

18. **Sidebar active detection trailing-slash mismatch** — Index routes not highlighted correctly. Fix: normalize trailing slash comparison.

19. **Enum display maps hardcoded English + unaccented Vietnamese** — Fix: replace with i18n translation keys.

### Design Issues (low priority, address if time permits)

20. **Double try-catch in handlers + BillingNotificationService** — Handler catch blocks are dead code. Fix: remove outer try-catch, trust interface contract.

21. **Architecture test assertion always true** — `>= 0` on count. Fix: set meaningful threshold or remove.

22. **isActive in Zod schema for create path** — Field not rendered or submitted on create. Fix: split schema or conditionally exclude.

### Claude's Discretion
- Test organization and naming conventions
- Order of fixes (prioritize by risk)
- Whether to batch related fixes into single plans or separate
