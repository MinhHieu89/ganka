**1. Excel upload file size limit** (`PharmacyApiEndpoints.cs`):
Add 10MB file size check before processing, matching the pattern in `WarrantyApiEndpoints.cs`:
```csharp
const long maxFileSizeBytes = 10 * 1024 * 1024;
if (file.Length > maxFileSizeBytes) return Results.BadRequest("File size must not exceed 10 MB.");
```

**2. ConfirmDrugCatalogImport server-side re-validation** (`ConfirmDrugCatalogImport.cs`):
Re-validate every row in the handler: non-empty Name, non-empty Unit, SellingPrice >= 0, MinStockLevel >= 0. Return validation errors for invalid rows. Also check for duplicate names within batch and against existing catalog items via repository.

**3. TryParseInt fractional check** (`ImportDrugCatalogFromExcel.cs`):
Add fractional check before casting double to int:
```csharp
if (value is double d) { if (d != Math.Floor(d)) { result = 0; return false; } result = (int)d; return true; }
```

**4. OsdiHub authorization** (`OsdiHub.cs`):
- Change `visitId` parameter from `string` to `Guid`
- Add authorization check: verify the authenticated user (from `Context.UserIdentifier`) has access to the visit before adding to group
- Inject `IVisitRepository` or a lightweight auth service into the hub

**5. UploadClinicLogo fixes** (`UploadClinicLogo.cs`, `SettingsApiEndpoints.cs`):
- Pass `file.Length` from endpoint into the command instead of calling `Stream.Length` in handler
- After successful blob upload, call settingsService to persist the blob URL to ClinicSettings
- Add magic-byte validation: read first 12 bytes, verify against declared content type (FF D8 FF for JPEG, 89 50 4E 47 for PNG, 52 49 46 46 for WebP), then rewind stream

**6. .xls rejection** (`ImportDrugCatalogFromExcel.cs`):
Replace `.xls => ExcelType.XLSX` with explicit rejection:
```csharp
".xls" => throw new NotSupportedException("Old .xls format is not supported. Please save as .xlsx.")
```
Handle in the catch block with a user-friendly message.
