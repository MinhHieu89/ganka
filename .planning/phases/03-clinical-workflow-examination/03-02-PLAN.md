- Query: GetVisitByIdQuery
- Handler: Fetch via repository.GetByIdWithDetailsAsync, map to VisitDetailDto, return directly (no Result wrapper for queries)

GetActiveVisits.cs:
- Query: GetActiveVisitsQuery
- Handler: Fetch via repository.GetActiveVisitsAsync, map to List of ActiveVisitDto
- ActiveVisitDto includes HasAllergies (bool) and WaitMinutes (int, calculated from VisitDate to now)
- For HasAllergies: include a denormalized HasAllergies boolean on Visit entity, set at creation time from patient data. This avoids cross-module queries at dashboard poll time.

SignOffVisit.cs:
- Command: SignOffVisitCommand
- Handler: Fetch visit by ID, call visit.SignOff(currentUserId), save. Returns Result

AmendVisit.cs:

- Validator: Reason required and not empty, FieldChangesJson required
- Handler: Fetch visit, verify Status == Signed, call visit.StartAmendment(), create VisitAmendment record, add to visit.Amendments, save. Returns Result

AdvanceWorkflowStage.cs:

- Handler: Fetch visit, call visit.AdvanceStage(newStage), save. Returns Result

UpdateVisitNotes.cs:

**Handlers:**

UpdateVisitRefraction.cs:
- Command: UpdateRefractionCommand (VisitId, Type, OdSph, OdCyl, OdAxis, OdAdd, OdPd, OsSph, OsCyl, OsAxis, OsAdd, OsPd, UcvaOd, UcvaOs, BcvaOd, BcvaOs, IopOd, IopOs, IopMethod, AxialLengthOd, AxialLengthOs)
- Validator: VisitId required. SPH in -30..+30, CYL in -10..+10, AXIS in 1..180, ADD in 0.25..4.0, PD in 20..80, VA in 0.01..2.0, IOP in 1..60, AxialLength in 15..40 (all nullable, only validated if provided)
- Handler: Fetch visit with details, call EnsureEditable(), find existing refraction of same type (or create new via Refraction.Create), call refraction.Update(all fields), save

AddVisitDiagnosis.cs:
- Command: AddVisitDiagnosisCommand
- Validator: VisitId required, Icd10Code required, Laterality must be valid enum value
- Handler: Fetch visit, call EnsureEditable(). If Laterality == OU, create TWO VisitDiagnosis records: one with laterality OD (code suffix .1 for right), one with OS (code suffix .2 for left). If RequiresLaterality is true for the base code, block unspecified (.9) -- but since our Laterality enum doesn't include Unspecified, this is enforced by type system. Set Role: if no existing diagnoses, Primary; else Secondary. Save.
- NOTE: The handler needs to query ICD-10 code to check RequiresLaterality. Inject ReferenceDbContext for this lookup.

RemoveVisitDiagnosis.cs:
- Command: RemoveVisitDiagnosisCommand (VisitId, DiagnosisId)
- Handler: Fetch visit, call EnsureEditable(), remove diagnosis by Id, save

SearchIcd10Codes.cs:
- Query: SearchIcd10CodesQuery (SearchTerm, DoctorId?)
- Handler: Inject ReferenceDbContext. Query Icd10Codes where Code LIKE '%term%' OR DescriptionEn LIKE '%term%' OR DescriptionVi LIKE '%term%'. If DoctorId provided, also query DoctorIcd10Favorites to mark IsFavorite=true on matching codes. Order: favorites first, then alphabetical by Code. Map to List of Icd10SearchResultDto.

ToggleIcd10Favorite.cs:
- Command: ToggleIcd10FavoriteCommand (DoctorId, Icd10Code)
- Handler: Check if favorite exists. If yes, remove. If no, add. Save. Returns Result.

GetDoctorFavorites.cs:
- Query: GetDoctorFavoritesQuery (DoctorId)
- Handler: Get favorites via repository, query ReferenceDbContext for full code details, return List of Icd10SearchResultDto with IsFavorite=true.
