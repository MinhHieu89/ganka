**Icd10Combobox.tsx:**
- Follows EXACTLY the Popover + Command pattern from AllergyForm combobox:
  - Popover with Button trigger (not Input) wrapped in div (avoid click-to-toggle anti-pattern)
  - Command with shouldFilter={false} (external filtering via API)
  - CommandInput for search text
  - CommandGroup for category grouping
  - CommandItem for each result with check icon for selected
- Uses useIcd10Search(searchTerm) hook with debounced search term (300ms delay)
- Search triggers when searchTerm.length >= 2
- Results grouped by Category (e.g., "Dry Eye", "Myopia", "Glaucoma")
- Per-doctor favorites: show starred items at top in a separate "Favorites" CommandGroup
  - Star icon (IconStar from @tabler/icons-react) toggleable on each result
  - Clicking star calls useToggleIcd10Favorite mutation
- Show bilingual: code + DescriptionVi + DescriptionEn (secondary text)
- Laterality enforcement:
  - When user selects a code where RequiresLaterality=true, show an inline laterality selector BEFORE adding
  - Laterality selector: three radio-like buttons: OD (Right) | OS (Left) | OU (Both)
  - If OU selected, the handler creates two records (.1 + .2) -- frontend just sends Laterality=2 (OU)
  - No "Unspecified" option (laterality enum has no .9 value)
  - If RequiresLaterality=false, skip the laterality selector and add directly
- On selection: call useAddDiagnosis mutation with the selected code and laterality
- Close Popover after selection

**SignOffSection.tsx:**
- Positioned at the bottom of the visit detail page
- When visit.status === 0 (Draft) or 2 (Amended):
  - Show "Sign Off Visit" Button (variant: default, full width)
  - On click: open AlertDialog with warning message from i18n (visit.signOffConfirm: "This will lock the record. Corrections will require an amendment.")
  - On confirm: call useSignOffVisit mutation
  - On success: page re-fetches visit data, all sections switch to read-only mode
- When visit.status === 1 (Signed):
  - Show signed badge with SignedAt timestamp
  - Show "Amend" Button (variant: outline)
  - On "Amend" click: open AmendmentDialog

**AmendmentDialog.tsx:**
- Dialog component that opens when doctor clicks "Amend" on a signed visit
- Contains:
  - Mandatory Textarea for amendment reason (minimum 10 characters via Zod validation)
  - "Confirm Amendment" button
- On confirm:
  - Call useAmendVisit mutation with { visitId, reason, fieldChangesJson: "[]" }
  - The actual field-level diff is computed on the BACKEND when the amended visit is later re-signed (the handler compares before/after snapshots)
  - On success: visit status changes to Amended, fields become editable again
  - Close dialog

**VisitAmendmentHistory.tsx:**
- Collapsible card section showing amendment audit trail
- Only visible when visit has amendments (amendments.length > 0)
- Default collapsed
- Lists each amendment:
  - AmendedByName, AmendedAt (formatted datetime)
  - Reason text
  - Field changes: parse FieldChangesJson and display as a table: Field | Old Value | New Value
- Most recent amendment first (reverse chronological)
- Read-only display (no editing)
