```typescript

```

```typescript
type: number

The `visitId` field can also be removed from the interface since the backend RefractionDto record does not include it (check the C# record -- it has Id, Type, then measurement fields, no VisitId). IMPORTANT: Do NOT change the `updateRefraction` function or `useUpdateRefraction` hook. They correctly send `refractionType` in the request body, which the backend `UpdateRefractionCommand` expects. The write path is correct; only the read path (DTO) is broken.

**Fix 2 -- RefractionSection.tsx: Update getRefractionByType to use `r.type` (line 43)**

Change:
```typescript
refractions.find((r) => r.refractionType === type)
```
To:
```typescript
refractions.find((r) => r.type === type)
```

This fixes both the refraction data loading after reload AND the tab (*) data indicator (both depend on this lookup).

**Fix 3 -- RefractionForm.tsx: Fix IOP Select controlled/uncontrolled warning**

The current code (lines 287-290) returns `undefined` when iopMethod is null, making the Select uncontrolled. When data loads and iopMethod becomes a number, it switches to controlled, triggering the React warning.

Fix: Make the Select always controlled by using empty string `""` as the "no selection" value instead of `undefined`:

Change the value prop (lines 287-290):
```typescript
value={(() => {
  const v = form.watch("iopMethod") as number | null | undefined
  return v === null || v === undefined ? undefined : String(v)
})()}
```
To:
```typescript
value={(() => {
  const v = form.watch("iopMethod") as number | null | undefined
  return v === null || v === undefined ? "" : String(v)
})()}
```

This keeps the Select always in controlled mode (always receives a string). Shadcn Select with value="" renders the placeholder, same as before visually.
