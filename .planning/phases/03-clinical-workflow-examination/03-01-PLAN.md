**EF Core Configurations:**

VisitConfiguration:
- ToTable("Visits", "clinical")
- HasKey(Id), RowVersion as concurrency token with IsRowVersion()
- PatientId, DoctorId as required Guid
- PatientName, DoctorName as required strings (maxlength 200)
- CurrentStage as int conversion, Status as int conversion
- ExaminationNotes as optional nvarchar(max)
- HasMany(Refractions) with FK VisitId, cascade delete
- HasMany(Diagnoses) with FK VisitId, cascade delete
- HasMany(Amendments) with FK VisitId, cascade delete
- Index on PatientId, Index on DoctorId, Index on (CurrentStage, Status) for dashboard queries
- BranchId configured per AggregateRoot pattern

VisitAmendmentConfiguration:
- ToTable("VisitAmendments", "clinical")
- Reason required (maxlength 2000), FieldChangesJson required (nvarchar max)

RefractionConfiguration:
- ToTable("Refractions", "clinical")
- Type as int conversion
- All decimal? fields with precision(5,2) for diopter/VA values, AxialLength precision(5,2)
- IopMethod as nullable int conversion
- Unique index on (VisitId, Type) -- one refraction per type per visit

VisitDiagnosisConfiguration:
- ToTable("VisitDiagnoses", "clinical")
- Icd10Code required string (maxlength 20), DescriptionEn/DescriptionVi as required strings
- Laterality and Role as int conversions
- Index on (VisitId, Icd10Code, Laterality) unique -- prevent duplicate diagnosis+laterality

DoctorIcd10FavoriteConfiguration:
- ToTable("DoctorIcd10Favorites", "clinical")

- Unique index on (DoctorId, Icd10Code)
