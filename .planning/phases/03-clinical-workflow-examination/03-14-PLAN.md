  <action>
Fix the renderNumberInput function in RefractionForm.tsx to preserve decimal points during typing.

The current bug: `onChange` calls `Number(val)` on every keystroke. `Number("1.")` === 1, so the decimal point is immediately erased from the controlled input before the user can type the fractional digit.

The fix approach — use a local string state for in-progress typing, coerce to Number only on blur:

1. In `renderNumberInput`, add a `useState<string>` for the raw input string. Initialize it from `toFormValue(value)`.

2. Add a `useEffect` that syncs the local string state when the form value changes externally (e.g., after save/reload). Only update local state if the form value (converted to string) differs from what would be produced by `Number(localState)` — this avoids overwriting in-progress typing.

3. Change `onChange` to ONLY update the local string state (no `Number()` call, no `form.setValue`). Still clear server errors if present.

4. Change `onBlur` to coerce the local string to a Number and call `form.setValue(name, ...)`. Use the existing `handleBlur` plus the new coercion:
   ```typescript
   const numVal = localValue === "" ? null : Number(localValue)
   // Only update form if it's a valid number (not NaN)
   if (localValue !== "" && isNaN(numVal as number)) {
     setLocalValue(toFormValue(value as number | null | undefined))
   } else {
     form.setValue(name, numVal as any)
   }
   handleBlur?.()
   ```

5. The `<Input>` value prop changes from `toFormValue(value)` to the local string state.

6. Keep `type="number"` on the input — browsers with `type="number"` and `step` attributes handle decimal display natively. However, if `type="number"` still causes issues with the local string state approach (some browsers fire onChange with coerced values), change to `type="text"` with `inputMode="decimal"` and `pattern="[0-9]*[.]?[0-9]*"` to get the numeric keyboard on mobile while preserving raw string values.

IMPORTANT: The `renderNumberInput` function is called inside a loop for each field. The local state must be per-field. Since `renderNumberInput` is NOT a React component (it's a render function), you cannot use hooks inside it. Instead, either:
  - (a) Extract it into a proper React component (e.g., `NumberInput`) that accepts `name`, `field`, `disabled`, `form` as props and uses its own `useState`, OR
  - (b) Use an uncontrolled input with a `ref` and `defaultValue`, syncing on blur.

Option (a) is recommended — extract a `NumberInput` component at the top of the file (or inline as a named function component).
