  <behavior>
    UploadMedicalImage:
    - Valid image file (JPEG, 5MB) -> uploads to Azure Blob, creates MedicalImage entity, returns image ID
    - Valid video file (MP4, 150MB) -> uploads with extended size limit, returns image ID
    - File too large (image > 50MB) -> validation error
    - File too large (video > 200MB) -> validation error
    - Invalid content type (exe) -> validation error
    - Visit not found -> Result.Failure
    - Image uploaded after visit sign-off -> SUCCESS (images are append-only, no EnsureEditable check)

    GetVisitImages:
    - Visit with 5 images -> returns 5 MedicalImageDto with SAS URLs
    - Visit with no images -> returns empty list
    - SAS URLs have 1-hour expiry

    GetImageComparisonData:
    - Two visits, same patient, same image type -> returns both visits' images of that type
    - Visits belong to different patients -> Result.Failure
    - One visit has no images of requested type -> returns empty list for that visit

    DeleteMedicalImage:
    - Valid image ID -> deletes from blob storage + DB, returns success
    - Image not found -> Result.Failure
    - Only the uploader or a doctor can delete (basic auth check)
  </behavior>
  <action>
    Follow strict TDD: write failing tests first, then implement handlers.

    **UploadMedicalImage handler:**
    - Validator: file size (images max 50MB, video max 200MB based on ImageType), content type whitelist (JPEG, PNG, TIFF, BMP, WebP for images; MP4, MOV, AVI for video), VisitId required
    - Handle: verify visit exists (GetByIdAsync, NOT GetByIdWithDetailsAsync -- no need for full load), generate blob name (`clinical-images/{visitId}/{timestamp}_{sanitizedFileName}`), upload via IAzureBlobService.UploadAsync, create MedicalImage entity, save via IMedicalImageRepository.AddAsync + IUnitOfWork
    - IMPORTANT: Do NOT check EnsureEditable -- images are append-only even after sign-off
    - Set UploadedById from ICurrentUser

    **GetVisitImages handler:**
    - Query IMedicalImageRepository.GetByVisitIdAsync
    - For each image, generate SAS URL via IAzureBlobService.GetSasUrlAsync with 1-hour expiry
    - Map to MedicalImageDto (id, type, eyeTag, fileName, url, contentType, fileSize, description, createdAt)

    **GetImageComparisonData handler:**
    - Takes patientId, visitId1, visitId2, imageType
    - Verify both visits belong to patientId (security check)
    - Query IMedicalImageRepository.GetByVisitIdAndTypeAsync for each visit
    - Generate SAS URLs for all images
    - Return ImageComparisonResponse with visit1Images and visit2Images

    **DeleteMedicalImage handler:**
    - Find image by ID, delete blob via IAzureBlobService.DeleteAsync, remove from DB
  </action>

  </verify>
