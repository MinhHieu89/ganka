  <behavior>
    UpdateDryEyeAssessment:
    - Valid command with all fields -> creates/updates assessment, returns Result.Success
    - Visit not found -> returns Result.Failure with "Visit not found" error
    - Visit is signed -> returns Result.Failure (EnsureEditable guard)
    - Find-or-create pattern: if assessment exists for visit, update it; otherwise create new + add via domain method + AddDryEyeAssessment on repo

    OSDI Calculation:
    - All 12 questions answered (all 4s) -> score = 100.0, severity = Severe
    - All 12 questions answered (all 0s) -> score = 0.0, severity = Normal
    - Mixed: [4,4,4,4,4, 0,0,0,0, null,null,null] -> answered=9, sum=20, score=(20*100)/(9*4)=55.56, severity=Severe
    - Score 15 -> Mild, Score 25 -> Moderate
    - 0 questions answered -> score = null, no error (guard: answered==0 returns null)
    - Single question answered (value=2) -> score = (2*100)/(1*4) = 50.0

    SubmitOsdiQuestionnaire:
    - Valid token, not expired, 12 answers -> calculates score, saves OsdiSubmission, updates DryEyeAssessment OSDI fields
    - Expired token -> Result.Failure with "Token expired"
    - Invalid token -> Result.Failure with "Token not found"
    - At least 1 answered question required, validation error if all null

    GenerateOsdiLink:
    - Valid visitId -> creates OsdiSubmission with cryptographically secure token, 24-hour expiry, returns token + URL
    - Visit not found -> Result.Failure

    GetOsdiByToken:
    - Valid token, not expired -> returns questionnaire DTO with 12 questions (Vietnamese + English text)
    - Expired token -> Result.Failure
    - Invalid token -> Result.Failure
  </behavior>
  <action>
    Follow strict TDD: write failing tests first, then implement handlers.

    **UpdateDryEyeAssessment handler:**
    - Static class with Validator (FluentValidation) and Handle method
    - Validator: MeibomianGrading 0-3, Staining 0-5, TBUT >= 0, Schirmer >= 0, TearMeniscus >= 0
    - Handle: find visit via repo, EnsureEditable, find-or-create DryEyeAssessment (same pattern as UpdateRefraction), save via UnitOfWork
    - Follow the existing UpdateVisitRefraction pattern exactly

    **OSDI score calculation utility:**
    - Create a static OsdiCalculator class in Clinical.Application (or co-located with handler)
    - Formula: (sum * 100) / (answered * 4) where answered = count of non-null answers
    - Guard: if answered == 0, return null
    - Severity: Normal 0-12, Mild 13-22, Moderate 23-32, Severe 33+
    - Used by both SubmitOsdiQuestionnaire and UpdateDryEyeAssessment

    **SubmitOsdiQuestionnaire handler:**
    - Looks up OsdiSubmission by token, checks expiry
    - Calculates OSDI score using OsdiCalculator
    - Updates OsdiSubmission with answers + score
    - Also updates DryEyeAssessment.OsdiScore on the visit (create DryEyeAssessment if not exists)
    - No authentication required (public endpoint)

    **GenerateOsdiLink handler:**
    - Creates OsdiSubmission with PublicToken = Convert.ToBase64String(RandomNumberGenerator.GetBytes(32))
    - TokenExpiresAt = DateTime.UtcNow.AddHours(24)
    - Returns token and constructed URL

    **GetOsdiByToken handler:**
    - Queries OsdiSubmission by PublicToken
    - Checks TokenExpiresAt > DateTime.UtcNow
    - Returns 12 OSDI questions with both Vietnamese and English text
    - Returns minimal data: questions + visit date only (no patient name for security)
  </action>
