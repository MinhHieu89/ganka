**DrugPrescriptionSection.tsx**: Complete drug prescription section:

Props: `visitId: string`, `patientId: string`, `drugPrescriptions: DrugPrescriptionDto[]`, `disabled: boolean` (read-only after sign-off)

Structure:
- **VisitSection** wrapper with title "Don thuoc" (Drug Prescription), defaultOpen
- **headerExtra**: "Them thuoc" (Add Drug) button (hidden when disabled/read-only)
- **Prescription items list**: Table or card list showing each drug item:
  - Drug name (bold) + generic name (muted) + strength
  - Form + Route (translated via enum maps)
  - Dosage: show DosageOverride if present, otherwise Dosage (hybrid display)
  - Quantity + Unit
  - Frequency + Duration
  - IsOffCatalog badge ("Ngoai danh muc")
  - HasAllergyWarning: destructive Badge ("Di ung") if flagged
  - Edit/Remove action buttons (hidden when disabled)
- **Loi dan (Doctor's advice)** section at bottom: editable Textarea for notes
- **Empty state**: "Chua co don thuoc" (No prescriptions yet) with add button

Interaction flow (add-then-edit per locked decision):
1. Doctor clicks "Them thuoc" -> DrugPrescriptionForm dialog opens
2. Doctor selects drug from catalog (or enters off-catalog) -> fills dosage fields
3. If allergy match detected: DrugAllergyWarning banner shows inline in form
4. Doctor clicks "Them" (Add) -> item added to local state
5. Doctor can add more items or edit/remove existing
6. When all items ready, calls useAddDrugPrescription mutation with all items + notes

**Blocking AlertDialog** (locked decision):
- When doctor submits prescription that contains items with HasAllergyWarning=true
- Show AlertDialog: "Canh bao di ung" title, list allergies, "Xac nhan ke don" vs "Huy" buttons
- Doctor must explicitly confirm to proceed (or cancel)
- Uses AlertDialog component (non-dismissible, per established pattern)

Patient allergies: fetched via visit.patientId to query patient allergies. Either pass as prop from VisitDetailPage or fetch within section.
