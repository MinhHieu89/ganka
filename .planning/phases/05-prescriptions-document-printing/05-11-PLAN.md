**DocumentService.cs** (Clinical.Infrastructure/Services/):

- GenerateDrugPrescriptionAsync:
  1. Load visit with drug prescriptions and diagnoses
  2. Load clinic settings
  3. Build DrugPrescriptionData record with all needed info
  4. Create DrugPrescriptionDocument, call GeneratePdf(), return bytes
- Other methods: stub with NotImplementedException for now (implemented in Plan 12)
- Register in Clinical.Infrastructure IoC.cs: `services.AddScoped<IDocumentService, DocumentService>();`

**DrugPrescriptionDocument.cs** (Clinical.Infrastructure/Documents/):
- Implements QuestPDF `IDocument`
- Constructor takes DrugPrescriptionData + ClinicHeaderData
- A5 page size (148 x 210mm), 8mm margins, Noto Sans font 9pt body
- Layout per MOH Circular 26/2025 requirements:
  - **Header**: ClinicHeaderComponent (logo, name, address, phone, license)
  - **Title**: "DON THUOC" centered, bold 12pt
  - **Patient info**: Ho ten, Ngay sinh, Gioi tinh, Dia chi, CCCD
  - **Diagnosis**: Chan doan (from visit diagnoses)
  - **Drug table**: STT | Ten thuoc (drug name + generic + strength) | Cach dung (dosage/dosageOverride) | SL (quantity) | Don vi (unit)
  - **Loi dan** (Doctor's advice): notes field
  - **Footer**: Ngay tai kham (follow-up date, if provided) | Ngay ky (date) + Bac si kham benh (doctor name + signature line)
- Use DosageOverride if present, otherwise fall back to Dosage (hybrid dosage per locked decision)
- All text in Vietnamese with proper diacritics

Create a `DrugPrescriptionData` record holding all data needed for the document (patient name, DOB, gender, address, diagnoses, prescription items, notes, doctor name, prescription date).
