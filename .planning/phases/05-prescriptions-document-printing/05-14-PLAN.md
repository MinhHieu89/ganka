**prescription-api.ts**: TanStack Query hooks for prescriptions:
- `prescriptionKeys` factory: `{ drug: (visitId) => ['prescriptions', 'drug', visitId], optical: (visitId) => ['prescriptions', 'optical', visitId] }`
- `useAddDrugPrescription()`: useMutation for POST /api/clinical/{visitId}/drug-prescriptions
- `useUpdateDrugPrescription()`: useMutation for PUT
- `useRemoveDrugPrescription()`: useMutation for DELETE
- `useAddOpticalPrescription()`: useMutation for POST /api/clinical/{visitId}/optical-prescription
- `useUpdateOpticalPrescription()`: useMutation for PUT
- `useCheckDrugAllergy(patientId, drugName, genericName)`: useQuery for GET check-drug-allergy endpoint
- All mutations invalidate visit query cache on success (visitId key)
- All mutations have onError with toast.error

- Off-catalog option: "Add off-catalog drug" item at bottom of list (opens free-text input mode)
- When off-catalog selected, switch to Input mode where doctor types drug name manually
