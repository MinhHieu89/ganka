<feature>
  <name>Session Recording and Due Soon</name>
  <files>
    backend/src/Modules/Treatment/Treatment.Application/Features/RecordTreatmentSession.cs
    backend/src/Modules/Treatment/Treatment.Application/Features/GetTreatmentSessions.cs
    backend/src/Modules/Treatment/Treatment.Application/Features/GetDueSoonSessions.cs
    backend/tests/Treatment.Unit.Tests/Features/SessionHandlerTests.cs
  </files>
  <behavior>
    RecordTreatmentSession:
    - Valid command on active package -> records session, returns TreatmentSessionDto with optional IntervalWarning
    - Package not found -> NotFound error
    - Package not Active -> validation error (cannot record on Paused/Completed/etc.)
    - All sessions completed after recording -> package status auto-changes to Completed (TRT-04)
    - Session within minimum interval -> returns warning object but still records (TRT-05, soft enforcement)
    - Session with override reason -> stores IntervalOverrideReason
    - Consumables list -> stored as SessionConsumable entities, included in TreatmentSessionCompletedEvent
    - OSDI score recorded -> stored on session, severity calculated (TRT-03)

    GetTreatmentSessions:
    - Package with sessions -> returns ordered list
    - Package with no sessions -> returns empty list
    - Non-existent package -> NotFound error

    GetDueSoonSessions:
    - Active packages with last session past min interval -> included in results
    - Active packages with no sessions yet -> included (never started, due immediately)
    - Active packages with recent session (within interval) -> excluded
    - No active packages -> empty result
  </behavior>
  <implementation>
    RecordTreatmentSession.cs:
    - Command: PackageId, ParametersJson, OsdiScore?, OsdiSeverity?, ClinicalNotes?, VisitId?, IntervalOverrideReason?, Consumables (list of {ConsumableItemId, ConsumableName, Quantity})
    - Validator: PackageId NotEmpty, ParametersJson NotEmpty
    - Handler flow:
      1. Load package with sessions
      2. Check interval: if last session exists and days since < MinIntervalDays, build warning (but don't block)
      3. Call package.RecordSession() -- domain handles status, events
      4. Save via UnitOfWork (Wolverine outbox dispatches events)
      5. Map session to DTO, include warning if applicable
    - Return type: Result of RecordSessionResponse(TreatmentSessionDto Session, IntervalWarning? Warning)
    - IntervalWarning record: (int DaysSinceLast, int MinIntervalDays)

    GetTreatmentSessions.cs:
    - Query: PackageId (Guid)
    - Handler: load package, return mapped sessions ordered by SessionNumber

    GetDueSoonSessions.cs:
    - Query: no required params
    - Handler: calls repo.GetDueSoonAsync(), maps packages to DTOs with NextDueDate computed
    - Returns list of TreatmentPackageDto (lightweight, with last session date and computed next due)
  </implementation>
</feature>
