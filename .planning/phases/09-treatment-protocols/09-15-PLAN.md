<feature>
  <name>Cancellation Approval Workflow</name>
  <files>
    backend/src/Modules/Treatment/Treatment.Application/Features/RequestCancellation.cs
    backend/src/Modules/Treatment/Treatment.Application/Features/ApproveCancellation.cs
    backend/src/Modules/Treatment/Treatment.Application/Features/RejectCancellation.cs
    backend/src/Modules/Treatment/Treatment.Application/Features/GetPendingCancellations.cs
    backend/tests/Treatment.Unit.Tests/Features/CancellationHandlerTests.cs
  </files>
  <behavior>
    RequestCancellation:
    - Active package + reason -> sets status to PendingCancellation, creates CancellationRequest
    - Non-existent package -> NotFound error
    - Already pending/cancelled/completed -> error
    - Empty reason -> validation error

    ApproveCancellation:
    - Pending package + valid manager PIN + deduction% -> approves, calculates refund, sets status Cancelled
    - Invalid manager PIN -> error (cross-module VerifyManagerPinQuery returns false)
    - DeductionPercent outside 10-20 -> validation error
    - Package not in PendingCancellation -> error
    - Refund calculation: uses package.CalculateRefundAmount(deductionPercent)

    RejectCancellation:
    - Pending package + rejection reason -> rejects, sets status back to Active
    - Package not in PendingCancellation -> error
    - Empty rejection reason -> validation error

    GetPendingCancellations:
    - Returns all packages with PendingCancellation status
    - Includes CancellationRequest details (requester, reason, date)
    - Empty list when no pending cancellations
  </behavior>
  <implementation>
    RequestCancellation.cs:
    - Command: PackageId, Reason
    - Handler: loads package, calls package.RequestCancellation(reason, requestedById), saves
    - Sets default deduction from protocol template's CancellationDeductionPercent

    ApproveCancellation.cs:
    - Command: PackageId, ManagerId, ManagerPin, DeductionPercent
    - Handler:
      1. Verify manager PIN via IMessageBus.InvokeAsync(new VerifyManagerPinQuery(ManagerId, ManagerPin))
      2. Load package (check PendingCancellation status)
      3. Calculate refund = package.CalculateRefundAmount(DeductionPercent)
      4. Call package.ApproveCancellation with manager info, deduction, refund amount
      5. Save

    RejectCancellation.cs:
    - Command: PackageId, ManagerId, RejectionReason
    - Handler: loads package, calls package.RejectCancellation(managerId, reason), saves

    GetPendingCancellations.cs:
    - Query: no params
    - Handler: calls repo.GetPendingCancellationsAsync(), maps to DTOs

    Tests: Mock IMessageBus for VerifyManagerPinQuery. Test refund calculation for both pricing modes.
  </implementation>
</feature>
