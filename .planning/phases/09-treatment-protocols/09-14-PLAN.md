<feature>
  <name>Package Modification and Type Switching</name>
  <files>
    backend/src/Modules/Treatment/Treatment.Application/Features/ModifyTreatmentPackage.cs
    backend/src/Modules/Treatment/Treatment.Application/Features/SwitchTreatmentType.cs
    backend/src/Modules/Treatment/Treatment.Application/Features/PauseTreatmentPackage.cs
    backend/tests/Treatment.Unit.Tests/Features/ModifyPackageHandlerTests.cs
  </files>
  <behavior>
    ModifyTreatmentPackage:
    - Valid modification on Active/Paused package -> creates version snapshot, applies changes, returns updated dto
    - Non-existent package -> NotFound error
    - Completed/Cancelled/Switched package -> error (not modifiable)
    - Version snapshot captures previous and current state as JSON
    - ChangeDescription generated from diff (e.g., "Session count changed from 4 to 6")

    SwitchTreatmentType:
    - Valid switch on Active/Paused package -> marks old as "Switched", creates new package from new template with remaining sessions
    - Non-existent package -> NotFound error
    - Non-existent new template -> NotFound error
    - Terminal status package -> error
    - New package inherits remaining session count (TotalSessions - SessionsCompleted)
    - Returns new TreatmentPackageDto

    PauseTreatmentPackage:
    - Active package + Pause action -> sets status to Paused
    - Paused package + Resume action -> sets status to Active
    - Active package + Resume -> error (already active)
    - Completed/Cancelled package + either action -> error
  </behavior>
  <implementation>
    ModifyTreatmentPackage.cs:
    - Command: PackageId, TotalSessions?, ParametersJson?, MinIntervalDays?, Reason (required)
    - Validator: PackageId NotEmpty, Reason NotEmpty, TotalSessions 1-6 if provided
    - Handler:
      1. Load package
      2. Build JSON snapshots of current state
      3. Call package.Modify() with changes and reason -- domain creates ProtocolVersion
      4. Generate ChangeDescription by comparing old vs new values
      5. Save, return updated DTO

    SwitchTreatmentType.cs:
    - Command: PackageId, NewProtocolTemplateId, Reason (required)
    - Handler:
      1. Load existing package (check status is modifiable)
      2. Load new protocol template (NotFound check)
      3. Calculate remaining sessions = existing.TotalSessions - existing.SessionsCompleted
      4. Call existing.MarkAsSwitched()
      5. Create new package from template with remaining sessions, link to same PatientId
      6. Save both changes in single UnitOfWork transaction
      7. Return new package DTO

    PauseTreatmentPackage.cs:
    - Command: PackageId, Action (enum: Pause/Resume)
    - Handler: Load package, call Pause() or Resume() based on action, save
  </implementation>
</feature>
