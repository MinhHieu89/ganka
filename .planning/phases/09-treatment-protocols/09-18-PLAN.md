  <behavior>
    DeductTreatmentConsumables:
    - Event with consumables list -> deducts each consumable by quantity from warehouse
    - ConsumableItem not found -> logs warning, continues with remaining items (don't fail entire deduction)
    - Insufficient stock -> logs warning, deducts whatever is available
    - Empty consumables list -> no-op (no error)
    - Multiple consumables -> deducts all in single transaction
  </behavior>
  <implementation>
    1. Update Pharmacy.Application.csproj:
       - Add ProjectReference to Treatment.Contracts (for TreatmentSessionCompletedEvent)
       - Treatment.Contracts must already exist (from plan 06)

    2. DeductTreatmentConsumables.cs:
       - namespace Pharmacy.Application.Features.Consumables
       - Static class DeductTreatmentConsumablesHandler
       - public static async Task Handle(TreatmentSessionCompletedEvent message, IConsumableRepository repository, IUnitOfWork unitOfWork, CancellationToken ct)
       - For each ConsumableUsage in message.Consumables:
         - Load consumable item by ID
         - If found, call item.RemoveStock(usage.Quantity) or use FEFO allocator for batch items
         - If not found, log warning and continue
       - Save all changes at end

    3. Tests:
       - Test deduction with valid consumables
       - Test deduction with missing consumable (graceful skip)
       - Test empty consumables list (no-op)
       - Test multiple consumables in single event
  </implementation>
</feature>
