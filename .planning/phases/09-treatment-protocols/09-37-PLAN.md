Replace the static stub handler with an async handler that uses IUserRepository:

```csharp
using Auth.Application.Interfaces;
using Auth.Contracts.Queries;

namespace Auth.Application.Features;

/// <summary>
/// Wolverine handler for cross-module VerifyManagerPinQuery.
/// Verifies the provided PIN against the stored hashed PIN for the manager.
/// </summary>
public static class VerifyManagerPinHandler
{
    public static async Task<VerifyManagerPinResponse> HandleAsync(
        VerifyManagerPinQuery query,
        IUserRepository userRepository,
        CancellationToken ct)
    {
        if (string.IsNullOrWhiteSpace(query.Pin))
            return new VerifyManagerPinResponse(false);

        var user = await userRepository.GetByIdAsync(query.ManagerId, ct);
        if (user is null)
            return new VerifyManagerPinResponse(false);

        var isValid = user.VerifyManagerPin(query.Pin);
        return new VerifyManagerPinResponse(isValid);
    }
}
```

Key changes:
- Method is now async (HandleAsync) -- Wolverine auto-discovers async handlers
- Injects IUserRepository via Wolverine's method injection
- Calls User.VerifyManagerPin() which uses BCrypt.Verify
- Returns false for null user, empty PIN, or missing PIN hash

Do NOT change the query/response records in Auth.Contracts -- all existing callers (ApproveCancellation, ApproveRefund, ApproveDiscount, RejectDiscount) use the same contract and will automatically benefit.
