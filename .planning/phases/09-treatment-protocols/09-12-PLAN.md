  <name>Treatment Package Creation and Queries</name>
  <files>
    backend/src/Modules/Treatment/Treatment.Application/Features/CreateTreatmentPackage.cs
    backend/src/Modules/Treatment/Treatment.Application/Features/GetPatientTreatments.cs
    backend/src/Modules/Treatment/Treatment.Application/Features/GetTreatmentPackageById.cs
    backend/src/Modules/Treatment/Treatment.Application/Features/GetActiveTreatments.cs
    backend/tests/Treatment.Unit.Tests/Features/TreatmentPackageHandlerTests.cs
  </files>
  <behavior>
    CreateTreatmentPackage:
    - Valid command with template ID -> creates package with template defaults, returns TreatmentPackageDto
    - Non-existent template ID -> NotFound error
    - Custom overrides (session count, price) -> uses overrides instead of template defaults
    - SessionCount > 6 or < 1 -> validation error
    - Sets status to Active, CreatedById from ICurrentUser

    GetPatientTreatments:
    - Patient with packages -> returns list with sessions completed/remaining
    - Patient with no packages -> returns empty list
    - Multiple active packages (TRT-06) -> returns all

    GetTreatmentPackageById:
    - Existing ID -> returns full package with sessions, versions, cancellation request
    - Non-existent ID -> NotFound error

    GetActiveTreatments:
    - Returns all active packages across patients
    - Empty result when no active packages
  </behavior>
  <implementation>
    CreateTreatmentPackage.cs:
    - Command: ProtocolTemplateId, PatientId, PatientName, TotalSessions?, PricingMode?, PackagePrice?, SessionPrice?, MinIntervalDays?, ParametersJson?, VisitId?
    - Nullable fields default to template values when not provided
    - Validator: ProtocolTemplateId NotEmpty, PatientId NotEmpty, PatientName NotEmpty, TotalSessions 1-6 if provided
    - Handler: loads template, creates package with defaults or overrides, adds to repo, saves
    - Maps to TreatmentPackageDto (Sessions empty, SessionsCompleted=0)

    GetPatientTreatments.cs:
    - Query: PatientId (Guid)
    - Also handle the cross-module GetPatientTreatmentsQuery from Treatment.Contracts
    - Handler: calls packageRepository.GetByPatientIdAsync, maps to DTOs
    - Computes LastSessionDate and NextDueDate from session data

    GetTreatmentPackageById.cs:
    - Query: PackageId (Guid)
    - Handler: calls repo.GetByIdAsync with includes, maps to full DTO

    GetActiveTreatments.cs:
    - Query: no params (or optional TreatmentType filter)
    - Handler: calls repo.GetActivePackagesAsync, maps to DTOs
  </implementation>
</feature>
