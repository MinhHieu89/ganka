  <name>Protocol Template CRUD</name>
  <files>
    backend/src/Modules/Treatment/Treatment.Application/Features/CreateProtocolTemplate.cs

    backend/tests/Treatment.Unit.Tests/Features/ProtocolTemplateHandlerTests.cs
  </files>
  <behavior>
    CreateProtocolTemplate:
    - Valid command -> creates template, returns TreatmentProtocolDto
    - SessionCount < 1 or > 6 -> validation error
    - DeductionPercent < 10 or > 20 -> validation error
    - Empty name -> validation error
    - Sets CreatedById from ICurrentUser

    UpdateProtocolTemplate:
    - Valid command + existing template -> updates, returns dto
    - Non-existent template ID -> NotFound error
    - Same validation as create for updated fields

    GetProtocolTemplates:
    - Returns list of active templates (optionally include inactive)
    - Filter by TreatmentType optional
    - Returns TreatmentProtocolDto list

    GetProtocolTemplateById:
    - Valid ID -> returns TreatmentProtocolDto
    - Non-existent ID -> NotFound error
  </behavior>
  <implementation>
    Each handler file follows the vertical slice pattern: Command/Query record + FluentValidation Validator class + static Handler class.

    CreateProtocolTemplate.cs:
    - Command: Name, TreatmentType (int), DefaultSessionCount, PricingMode (int), DefaultPackagePrice, DefaultSessionPrice, MinIntervalDays, MaxIntervalDays, DefaultParametersJson, CancellationDeductionPercent, Description
    - Validator: Name NotEmpty MaxLength(200), DefaultSessionCount 1-6, CancellationDeductionPercent 10-20, prices >= 0
    - Handler: validates, calls TreatmentProtocol.Create, adds to repo, saves, maps to DTO
    - Dependencies: ITreatmentProtocolRepository, IUnitOfWork, ICurrentUser, IValidator

    UpdateProtocolTemplate.cs:
    - Command: Id (Guid), plus same fields as Create
    - Handler: validates, gets by id (NotFound check), calls Update on entity, saves, maps to DTO

    GetProtocolTemplates.cs:
    - Query: TreatmentType? (int?), IncludeInactive (bool, default false)
    - Handler: calls repo GetAllAsync or GetByTypeAsync, maps to DTOs

    GetProtocolTemplateById.cs:
    - Query: Id (Guid)
    - Handler: calls repo GetByIdAsync, NotFound check, maps to DTO

    Tests: Use NSubstitute to mock repositories. Test each handler's happy path and error cases.
  </implementation>
</feature>
