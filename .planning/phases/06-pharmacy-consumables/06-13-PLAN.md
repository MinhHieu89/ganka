  <action>
    **DispenseDrugs.cs**:
    - Command: DispenseDrugsCommand(Guid PrescriptionId, Guid VisitId, Guid PatientId, string PatientName, DateTime PrescribedAt, string? OverrideReason, List<DispenseLineInput> Lines)
    - DispenseLineInput(Guid PrescriptionItemId, Guid? DrugCatalogItemId, string DrugName, int Quantity, bool IsOffCatalog, bool Skip, List<BatchOverride>? ManualBatches)
    - BatchOverride(Guid BatchId, int Quantity) -- for manual FEFO override
    - Validator: PrescriptionId required, at least one line
    - Handler:
      1. Check if already dispensed (GetByPrescriptionIdAsync)
      2. Check 7-day validity: if PrescribedAt + 7 days < now AND OverrideReason is null -> error
      3. Create DispensingRecord
      4. For each line:
         - If Skip -> AddLine with status Skipped, no batch deduction
         - If IsOffCatalog -> AddLine with status Dispensed, no batch deduction (manual drug)
         - Else: If ManualBatches provided -> use manual allocation; else -> call FEFOAllocator.Allocate with batches from IDrugBatchRepository.GetAvailableBatchesFEFOAsync
         - If allocation returns empty -> error (insufficient stock for {DrugName})
         - AddLine with status Dispensed, add BatchDeduction for each allocation
         - Call batch.Deduct(qty) for each allocated batch
      5. Save via repository + unit of work
      6. Return Result<Guid> with dispensing record ID

    **GetPendingPrescriptions.cs**: Query handler that routes to Clinical module via IMessageBus to get undispensed prescriptions, then enriches with dispensing status from Pharmacy.

    **GetDispensingHistory.cs**: Simple paginated query via IDispensingRepository.

    Run tests -- all should pass.
