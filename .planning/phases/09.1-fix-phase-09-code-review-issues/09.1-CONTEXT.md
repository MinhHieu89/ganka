### Priority Order
- Fix critical issues first (security, data integrity, broken functionality)
- Then important issues (logic errors, missing validations, i18n)

### Grouping Strategy
- Group fixes by layer to minimize context switching

- Infrastructure fixes standalone (migration regeneration)
- Frontend fixes grouped by component area

## Issues to Fix

### CRITICAL (13 issues)

#### Domain Layer
1. **TRT-05: Interval enforcement absent** — `TreatmentPackage.RecordSession()` never checks `MinIntervalDays` against last session date. Need to add guard clause with override reason support.
2. **TRT-02: Wrong session numbering** — `_sessions.Count + 1` counts cancelled sessions. Fix: `_sessions.Count(s => s.Status != SessionStatus.Cancelled) + 1`
3. **TRT-04/07: Modify() inconsistent state** — Setting `TotalSessions == SessionsCompleted` leaves `IsComplete=true` but `Status=Active`. Add auto-completion check in `Modify()`.
4. **TRT-09: No deduction range validation** — `RequestCancellation` accepts any deduction % (0-100). Add 10-20% guard.

#### Application Layer
5. **TRT-10: No permission authorization** — All endpoints use only `RequireAuthorization()`. Must add `.RequirePermissions()` on all write endpoints.
6. **TRT-09: ApproveCancellation ignores DeductionPercent** — Manager's override deduction is accepted but never used. Either implement recalculation or remove the parameter.
7. **TRT-10: RecordTreatmentSession trusts client PerformedById** — No `ICurrentUser` injection. Any user can impersonate a doctor.

#### Infrastructure
8. **Migration: BranchId missing** — `TreatmentProtocols` table in migration lacks `BranchId` column. Model snapshot has it. Runtime SqlException.

#### Frontend
9. **TRT-02/03/07/08/09: Non-functional buttons** — All action buttons in `TreatmentPackageDetail` have no `onClick` handlers. Dialogs exist but are never wired.
10. **TRT-03: Wrong JSON keys** — `TreatmentSessionCard` reads `pulses`/`zones`/`area` but form serializes `pulseCount`/`treatmentZones`/`treatmentArea`.
11. **TRT-10: No permission guards on routes** — Any authenticated user can access all treatment pages.
12. **TRT-03: Wrong QR URL format** — `SessionOsdiCapture` generates `/osdi/self-fill?token=` but route is `/osdi/$token`.
13. **TRT-03: QR token never registered** — `SessionOsdiCapture` generates UUID client-side but never POSTs to backend. QR scan always 404s.
