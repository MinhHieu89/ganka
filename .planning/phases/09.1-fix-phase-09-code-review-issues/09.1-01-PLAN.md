  <behavior>
    - RecordSession with interval violation and no override reason throws InvalidOperationException
    - RecordSession with interval violation and valid override reason succeeds
    - RecordSession with no prior sessions always succeeds (no interval to check)
    - Session numbering: after recording 3 sessions and cancelling session 2, next session is numbered 3 (not 4)
    - Modify(totalSessions: N) where N == completed count auto-sets Status=Completed and raises TreatmentPackageCompletedEvent
    - Modify(totalSessions: N) where N > completed count keeps Status=Active
    - Modify() with no actual changes does NOT create a ProtocolVersion snapshot
    - RequestCancellation with deductionPercent=5 throws (below 10%)
    - RequestCancellation with deductionPercent=25 throws (above 20%)
    - RequestCancellation with deductionPercent=15 succeeds
  </behavior>
  <action>
    1. In `RecordSession()`, add interval enforcement BEFORE creating the session:
       - Get last completed session (Status != Cancelled, ordered by CompletedAt descending)
       - If MinIntervalDays > 0 and last session exists, calculate days since last session using `scheduledAt ?? DateTime.UtcNow`
       - If days < MinIntervalDays and intervalOverrideReason is null/empty, throw InvalidOperationException with descriptive message
       - Add `string? intervalOverrideReason = null` parameter to RecordSession signature

    2. Fix session numbering in `RecordSession()`:
       - Replace `_sessions.Count + 1` with `_sessions.Count(s => s.Status != SessionStatus.Cancelled) + 1`

    3. In `Modify()`, add auto-completion check at the END of the method:
       - After setting TotalSessions, check `if (IsComplete && Status == PackageStatus.Active)`
       - If true, set `Status = PackageStatus.Completed` and `AddDomainEvent(new TreatmentPackageCompletedEvent(Id, PatientId, TreatmentType))`

    4. In `Modify()`, add change detection BEFORE creating version snapshot:
       - Compare new values with current values
       - Only create ProtocolVersion and raise ModifiedEvent if at least one value actually changed

    5. In `RequestCancellation()`, add deduction range guard:
       - `if (deductionPercent < 10 || deductionPercent > 20) throw new ArgumentOutOfRangeException(...)`

    6. Add domain events to `TreatmentProtocol.Update()`, `Deactivate()`, and `Activate()` methods (ProtocolUpdatedEvent, ProtocolDeactivatedEvent, ProtocolActivatedEvent) -- create simple record events if they don't exist.
  </action>

  </verify>
