## Decisions Made
- **Allowed Shared.Domain in Presentation:** The architecture test `Presentation_Should_Not_Depend_On_Domain_Directly` was updated to exclude `Shared.Domain` because `Result<T>` and `Error` are cross-cutting primitives that the Presentation layer legitimately needs for HTTP response mapping. Module-specific Domain layers (Auth.Domain, etc.) are still forbidden.
- **Combined Plan 03 + 04 deletions:** Since Plan 03 ran concurrently and had already deleted auth-flow files (IAuthService, AuthService, LoginEndpoint, etc.), Task 2 committed all deletions together rather than trying to separate them.
- **Added Logging.Abstractions package:** After Plan 03 removed FrameworkReference Microsoft.AspNetCore.App from Auth.Application.csproj, `ILogger<>` was no longer available. Added the lightweight logging abstractions package to support handler logging.
