## Decisions Made
- **Removed IAuditReadContext alias:** The backward-compat alias was only needed for old Wolverine.Http endpoints. With those deleted, the alias is unnecessary clutter. AuditDbContext now implements IAuditReadRepository directly.
- **Removed WolverineFx.Http from Audit.Application.csproj:** Plan 01 kept it because old endpoints depended on it. Now that endpoints are deleted, the Application layer achieves full HTTP independence per the restructuring goals.
- **Used [AsParameters] for GET query binding:** Cleaner than individual [FromQuery] parameters -- Minimal API binds all record properties from query string automatically.
- **ExportAuditLogsResponse contains byte[] + fileName:** Handler produces file contents, Presentation maps to Results.File(). No HTTP concerns leak into the handler.
