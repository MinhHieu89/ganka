**2. Create `Shared.Presentation/ResultExtensions.cs`:**

Static class `ResultExtensions` in namespace `Shared.Presentation` with two extension methods:

- `public static IResult ToHttpResult(this Result result)`:
  - If `result.IsSuccess` → `Results.Ok()`
  - If `result.IsFailure` → map `result.Error.Code`:
    - `"Error.Unauthorized"` → `Results.Problem(detail: result.Error.Description, title: "Unauthorized", statusCode: 401)`
    - `"Error.NotFound"` → `Results.NotFound()`
    - `"Error.Conflict"` → `Results.Conflict(new { error = result.Error.Description })`
    - `"Error.Validation"` → `Results.Problem(detail: result.Error.Description, title: "Validation Error", statusCode: 400)`
    - Default → `Results.Problem(result.Error.Description, statusCode: 400)`

- `public static IResult ToHttpResult<T>(this Result<T> result)`:
  - If `result.IsSuccess` → `Results.Ok(result.Value)`
  - If `result.IsFailure` → same error mapping as the non-generic version above

- `public static IResult ToCreatedHttpResult<T>(this Result<T> result, string routePrefix)`:
  - If `result.IsSuccess` → `Results.Created($"{routePrefix}/{result.Value}", new { Id = result.Value })`
  - If `result.IsFailure` → same error mapping as above
  - This handles the `POST` endpoints that return `201 Created` with a location header (e.g., CreateUser, CreateRole).

Analyze the current error mapping patterns in `AuthApiEndpoints.cs` to ensure ALL error codes used in existing endpoints are covered. The current endpoints use: `Error.Unauthorized`, `Error.NotFound`, `Error.Conflict`, and a default 400 fallback.

Using directives needed: `using Microsoft.AspNetCore.Http;`, `using Shared.Domain;`.

**3. Create `Shared.Presentation/HttpContextExtensions.cs`:**

Static class `HttpContextExtensions` in namespace `Shared.Presentation` with:

- `public static bool TryGetUserId(this HttpContext httpContext, out Guid userId)`:
  - Gets `ClaimTypes.NameIdentifier` from `httpContext.User.FindFirstValue(ClaimTypes.NameIdentifier)`
  - Returns `Guid.TryParse(claim, out userId)`
  - If claim is null or not a valid Guid, returns false and sets userId to `Guid.Empty`

Using directives: `using System.Security.Claims;`, `using Microsoft.AspNetCore.Http;`.
