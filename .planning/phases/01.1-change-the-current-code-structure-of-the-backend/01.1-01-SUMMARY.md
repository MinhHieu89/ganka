## Decisions Made
- **Kept WolverineFx.Http in Application csproj files:** Plan step 4 said to remove WolverineFx.Http and FrameworkReference from Auth.Application.csproj, but the Important Notes section said to NOT delete existing endpoints. Since those endpoints depend on WolverineFx.Http, the references must stay until Plans 03-04 migrate them. Added TODO comments in csproj files.
- **Backward-compatible interface aliases:** Created IAuditReadContext as an empty interface extending IAuditReadRepository, and Services.IJwtService as empty interface extending Interfaces.IJwtService. This allows existing code to compile unchanged while new code uses the canonical interfaces.
- **IPasswordHasher registered as Singleton:** Matching the existing PasswordHasher Singleton registration since the class is stateless (no injected dependencies).
- **Added Domain isolation test for Presentation:** Beyond the plan's request for Infrastructure isolation, also added a test ensuring Presentation doesn't reference Domain directly (should go through Application DTOs).
