## Decisions Made
- **Reflection-based navigation wiring:** Domain entities use private setters for navigation properties (EF Core manages these). Tests need fully wired entity graphs (user -> userRoles -> role -> rolePermissions -> permission) to test handler DTO mapping. Created TestHelpers methods using reflection to set these properties, avoiding the need to modify domain entities for testability.
- **Custom TestAsyncQueryable instead of MockQueryable NuGet:** Audit handlers use `IQueryable<T>.ToListAsync()` (EF Core extension). Rather than adding the MockQueryable.NSubstitute package, implemented a lightweight `TestAsyncEnumerable<T>` with `IAsyncQueryProvider` support. This avoids an additional dependency and is sufficient for the current handler test needs.
- **Two handler test patterns:** Instance handlers (Login, Logout, etc.) use a `CreateSut()` factory pattern with class-level mock fields. Static handlers (GetUsers, CreateUser, etc.) call `Handle()` directly passing mock dependencies as parameters, matching the Wolverine static handler convention.
