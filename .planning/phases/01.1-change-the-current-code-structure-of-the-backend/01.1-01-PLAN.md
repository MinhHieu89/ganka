<objective>
Create the foundational repository and service interfaces that all Auth feature handlers will depend on. This establishes the new data-access pattern (thin repository) and extracts the IPasswordHasher interface so that feature handlers can inject these abstractions instead of the old fat service interfaces.

Purpose: Without this foundation, no feature file can be created. The repository interface defines how handlers access data, and IPasswordHasher lets handlers verify passwords without depending on Infrastructure.

Output: IAuthRepository.cs, IPasswordHasher.cs, AuthRepository.cs, updated PasswordHasher.cs, updated Program.cs DI registrations.
</objective>
