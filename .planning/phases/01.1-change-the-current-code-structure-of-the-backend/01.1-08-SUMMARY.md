## Decisions Made
- **DbContext in Program.cs:** AuthDbContext and scaffold module DbContexts remain in Program.cs ConfigureDbContext helper because they need the cross-module AuditInterceptor singleton. Only AuditDbContext moved to Audit.Infrastructure IoC since it does NOT use AuditInterceptor.
- **connectionString parameter:** Audit.Infrastructure IoC accepts connectionString as a parameter so it fully owns AuditDbContext registration rather than splitting responsibility with Program.cs.
- **Registration order:** AddSharedInfrastructure() first (provides HttpContextAccessor), AddAuditInfrastructure() second (provides AuditInterceptor), AddAuthInfrastructure() third (consumes services from both).
- **FluentValidation.DependencyInjectionExtensions:** Added to Auth.Application.csproj since AddValidatorsFromAssembly lives in that package. Previously only referenced by Bootstrapper.
