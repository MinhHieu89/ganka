## Decisions Made
- **Used Contracts LoginResponse for handlers:** LoginHandler returns `Result<LoginResponse>` using `Auth.Contracts.Dtos.LoginResponse` since that's the canonical cross-module DTO. Removed the duplicate `LoginResponse` record from Login.cs feature file.
- **Separate RefreshTokenResponse type:** Defined `RefreshTokenResponse` in RefreshToken.cs as a semantically distinct type even though it has the same shape as LoginResponse.
- **Logout revokes all tokens:** Per plan specification, LogoutHandler uses `RevokeAllByUserIdAsync` to revoke all user sessions (not just a single token like the old AuthService).
- **Removed WolverineFx.Http from Auth.Application:** No more Wolverine.Http endpoints exist in Auth.Application, so the package reference and FrameworkReference are no longer needed.
- **Cleaned up IJwtService alias:** With AuthService deleted, the backward-compat `Auth.Application.Services.IJwtService` alias is no longer needed. JwtService now implements `Auth.Application.Interfaces.IJwtService` directly.
