### Localization approach
- Fix only the specific screens reported in TASK-4 (Cashier/Optical) and TASK-28 (Treatment templates) — no full i18n audit
- Add missing Vietnamese keys to existing `vi/*.json` locale files (established pattern: `frontend/public/locales/vi/{module}.json`)
- TASK-9 validation messages: backend returns error codes/keys, frontend translates via i18n — consistent with existing app pattern

### Image upload issues (TASK-8, TASK-10)
- Keep in scope but start with diagnosis — user needs to check deployment environment first
- Plans should begin with investigation of the upload failure before implementing fixes
- Azure Blob infrastructure already exists (`AzureBlobService.cs`, `UploadPatientPhoto.cs`, `UploadClinicLogo.cs`)

### Pharmacy/inventory stock logic
- stock=0 must show "Out of Stock" (not "Low Stock") — unified across medication and supply inventories
- stock=0 must trigger inventory alert — consistent behavior between medication and supply modules
- TASK-16 and TASK-23 (OTC catalog/search issues) treated as single root cause investigation
- TASK-15 (medication import) likely overlaps with Phase 10 plan 10-02 (Excel import with preview) — investigate if already addressed
