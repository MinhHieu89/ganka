## Typography

4 sizes, 2 weights only.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Page title | 22px | 700 (bold) | 1.2 | "Pre-Exam dashboard" h1 |
| KPI value | 24px | 700 (bold) | 1.2 | KPI card numeric values |
| Body | 14px | 400 (regular) | 1.5 | Table cells, banner info, general text, header technician name, patient name in table |
| Label / Sub-text | 12px | 400 (regular) | 1.4 | KPI labels, KPI sub-text, banner label, filter pill labels, "Loai" pill text |

**Weight scale:** 400 (regular) and 700 (bold). No 500 medium -- use bold for emphasis, regular for everything else.

**Merges applied (from 6-size / 3-weight original):**
- 13px user-name merged into 14px body (1px difference has no hierarchy value, both 400-weight secondary text)
- 11px hint merged into 12px label/sub-text (both small metadata)
- 500 medium removed: page title promoted to 700 bold, KPI labels and banner label demoted to 400 regular, patient name in table uses 400 regular (bold reserved for KPI values and page title only)

Source: technician-dashboard.md spec Sections 3, 4, 7.

## Color

### Surface Colors (60/30/10 split)

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `oklch(1 0 0)` (white) | Page background, table background |
| Secondary (30%) | `var(--card)` / `oklch(1 0 0)` | KPI cards, slide-over panel |
| Accent (10%) | `#534AB7` | Primary action button ("Tiep tuc do", "Nhan BN" menu item text) |
| Destructive | `#A32D2D` | Red flag menu item, red flag badge text, wait time >= 25min |

Accent reserved for: "Tiep tuc do" banner button background, "Nhan BN" dropdown menu item text color, primary action contexts only.

**Primary focal point:** In-progress banner (when visible); KPI cards (when no patient in progress). The banner's blue background and purple CTA button create the strongest visual pull. When the banner is absent, the four KPI cards with colored numeric values serve as the primary scan target.

## Component Inventory

### KPI Cards (4 cards, grid layout)

| Card | Icon | Label (vi) | Label (en) | Value Source | Sub-text (vi) |
|------|------|-----------|------------|-------------|---------------|
| 1 | `IconClock` | Cho kham | Waiting | COUNT(status=waiting) today | "trong hang doi" |
| 2 | `IconEye` | Dang do | In Progress | COUNT(status=in_progress, assigned=me) | "BN cua toi" |
| 3 | `IconCircleCheck` | Hoan tat Pre-Exam | Pre-Exam Completed | COUNT(status=completed) today | "hom nay" |
| 4 | `IconAlertTriangle` | Red flag | Red Flag | COUNT(is_red_flag=true) today | "fast-track hom nay" |

Layout: 4-column grid, gap 12px. Each card: label 12px regular, value 24px bold colored, sub-text 12px regular hint.

### Banner "Dang thuc hien Pre-Exam"

- Visibility: Only when current technician has 1 patient in "Dang do" status
- Content: label + patient name + birth year + chief complaint + check-in time
- Action: "Tiep tuc do" button (bg `#534AB7`, text white, navigates to stub Pre-Exam page)

### Toolbar

- Filter pills: Tat ca | Cho kham | Dang do | Hoan tat | Red flag (each shows count)
- Search input: placeholder "Tim BN theo ten, SDT...", debounce 300ms

### Patient Queue Table (9 columns)

| # | Column | Width | Content |
|---|--------|-------|---------|
| 1 | # | 32px | Row index |
| 2 | Ho ten | 22% | Patient name (font-weight 400 regular; red `#A32D2D` if red flag) |
| 3 | Sinh | 46px | Birth year (4 digits) |
| 4 | Check-in | 60px | HH:mm format |
| 5 | Cho | 54px | Wait time in minutes, updated every 60s client-side |
| 6 | Ly do kham | auto | Chief complaint (text-ellipsis; red if red flag) |
| 7 | Loai | 54px | "Moi" or "Tai kham" pill (12px, border, small) |
| 8 | Trang thai | 82px | Status badge (colored per status table above) |
| 9 | Actions | 40px | Vertical 3-dot icon, opens DropdownMenu |

Sort: FIFO by check-in time. "Dang do" pinned top. "Hoan tat" dimmed at 0.55 opacity.

### Action Menu (DropdownMenu, per status)

| Action | Available When | Style | Behavior |
|--------|---------------|-------|----------|
| Nhan BN | Cho kham | Text `#534AB7`, weight 700 | Accept + navigate to stub Pre-Exam |
| Tiep tuc do | Dang do (own) | Text `#534AB7`, weight 700 | Navigate to stub Pre-Exam |
| Hoan tat chuyen BS | Dang do | Normal + check icon | Complete order, advance visit |
| Tra lai hang doi | Dang do | Normal | Opens confirmation AlertDialog |
| --- separator --- | | | |
| Chuyen BS ngay | Dang do | Text `#A32D2D` (destructive) | Opens Red Flag AlertDialog |
| --- separator --- | | | |
| Xem ket qua | All statuses | Normal, always last | Opens Sheet slide-over from right |

### Red Flag Dialog (AlertDialog)

- Dropdown: reason selection (required)
  - Dau mat nhieu
  - Giam thi luc dot ngot
  - Trieu chung lech 1 ben ro
  - Khac (shows text input for custom reason)
- Confirm button: "Chuyen BS ngay" (destructive red)
- Cancel button: "Huy"
- Validation: Reason required; if "Khac" selected, custom text required

### Return to Queue Dialog (AlertDialog)

- Copy: "Tra BN {ten} ve hang doi? Du lieu da nhap se duoc giu nguyen."
- Confirm: "Xac nhan"
- Cancel: "Huy"

### Pause Current Patient Dialog (AlertDialog)

- Trigger: "Nhan BN" clicked while already measuring another patient
- Copy: "Ban dang do BN {ten}. Muon tam dung va nhan BN moi?"
- Confirm: "Tam dung & nhan BN moi"
- Cancel: "Huy"

### Patient Results Panel (Sheet, side="right")

- Read-only slide-over from right side
- Sections: Personal info, visit history, pre-exam data collected (if any)
- Technician cannot edit -- view only
- Queue table remains partially visible behind overlay

### Stub Pre-Exam Page

- Route: `/pre-exam` (or `/pre-exam/:orderId`)
- Content: Centered placeholder text "Man hinh Pre-Exam dang duoc phat trien" with back button to dashboard
- Minimal design, no functional form

## Interaction States

### Loading

- KPI cards: 4 Skeleton rectangles matching card dimensions
- Table: 5 Skeleton rows with column-width rectangles
- Banner: Hidden during loading (no skeleton)

### Polling Intervals (per D-17)

| Data | Interval | Method |
|------|----------|--------|
| Dashboard rows | 15 seconds | TanStack Query `refetchInterval` |
| KPI cards | 30 seconds | TanStack Query `refetchInterval` |
| Wait time column | 60 seconds | Client-side `setInterval` |

### Optimistic Concurrency (per D-15)

- On accept race condition: Toast notification "BN da duoc nhan boi {ten}" via Sonner, auto-refresh table
- No pessimistic locking, no WebSocket

### Toast Notifications (Sonner)

| Event | Type | Message |
|-------|------|---------|
| Accept success | success | (navigate immediately, no toast needed) |
| Accept race lost | error | "BN da duoc nhan boi {ten}" |
| Complete success | success | "Da chuyen BN {ten} sang bac si" |
| Return success | info | "Da tra BN {ten} ve hang doi" |
| Red flag success | warning | "Da chuyen BN {ten} sang BS (Red flag)" |
