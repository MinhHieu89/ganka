<objective>
Build the complete Auth module -- domain entities, JWT authentication, login/refresh/logout endpoints, user/role/permission CRUD, and data seeding.

Purpose: Authentication is the gatekeeper. Every subsequent feature depends on users being authenticated with role-based claims. This plan delivers the full auth stack: password hashing (Argon2id), JWT generation with refresh token rotation, RBAC with granular permissions, 7 predefined roles with preset permission templates, and the root admin user seeded on first startup.

Output: A working auth system where POST /api/auth/login returns JWT tokens, users have roles with configurable permissions, and the system self-seeds with 7 roles and a root admin on first run.
</objective>

Build the Auth application layer (endpoints + validators) and infrastructure services:

**Application Services (interfaces in Auth.Application):**

1. `IAuthService`:
   - LoginAsync(LoginRequest) -> Result<LoginResponse> (uses custom Result<T> from Shared.Domain)
   - RefreshTokenAsync(RefreshTokenRequest) -> Result<LoginResponse>
   - LogoutAsync(Guid userId, string refreshToken) -> Result
   - GetUserByIdAsync(Guid) -> Result<UserDto>

2. `IJwtService`:
   - GenerateAccessToken(User user, IEnumerable<string> permissions) -> (string token, DateTime expiresAt)
   - GenerateRefreshToken() -> string
   - GetPrincipalFromExpiredToken(string token) -> ClaimsPrincipal?

**Wolverine.HTTP Endpoints (in Auth.Application/Endpoints/):**

3. `LoginEndpoint` -- `[WolverinePost("/api/auth/login")]`:
   - Accepts LoginRequest, injects IAuthService
   - Returns Result<LoginResponse>
   - No [Authorize] -- public endpoint

4. `RefreshTokenEndpoint` -- `[WolverinePost("/api/auth/refresh")]`:
   - Accepts RefreshTokenRequest, injects IAuthService
   - Returns Result<LoginResponse> with rotated refresh token
   - No [Authorize] -- uses refresh token for auth

5. `LogoutEndpoint` -- `[WolverinePost("/api/auth/logout")]`:
   - [Authorize], gets userId from claims, revokes refresh token
   - Returns Result

6. `UserEndpoints`:
   - `[WolverineGet("/api/admin/users")]` [Authorize] -- list all users with roles (paginated)
   - `[WolverinePost("/api/admin/users")]` [Authorize] -- create user
   - `[WolverinePut("/api/admin/users/{id}")]` [Authorize] -- update user
   - `[WolverinePut("/api/admin/users/{id}/roles")]` [Authorize] -- assign roles to user
   - `[WolverineGet("/api/auth/me")]` [Authorize] -- get current user info

7. `RoleEndpoints`:
   - `[WolverineGet("/api/admin/roles")]` [Authorize] -- list all roles with permissions
   - `[WolverinePost("/api/admin/roles")]` [Authorize] -- create custom role
   - `[WolverinePut("/api/admin/roles/{id}/permissions")]` [Authorize] -- update role permissions

8. `PermissionEndpoints`:
   - `[WolverineGet("/api/admin/permissions")]` [Authorize] -- list all permissions grouped by module

9. `LanguagePreferenceEndpoint`:
   - `[WolverinePut("/api/auth/language")]` [Authorize] -- update current user's language preference
   - Accepts { language: "vi" | "en" }

**FluentValidation Validators:**

10. `LoginCommandValidator`: Email required + valid email, Password required + min 8 chars
11. `CreateUserCommandValidator`: Email required + valid email, FullName required + 2-100 chars, Password required + min 8 chars, at least one RoleId
12. `CreateRoleCommandValidator`: Name required + 2-50 chars

**Infrastructure Services:**

13. `PasswordHasher`:
   - Use Konscious.Security.Cryptography for Argon2id hashing (add NuGet package to Auth.Infrastructure)
   - HashPassword(string password) -> string (returns base64 encoded hash with embedded salt)
   - VerifyPassword(string password, string hash) -> bool
   - Argon2id params: DegreeOfParallelism=4, MemorySize=65536 (64MB), Iterations=3
   - If Konscious causes issues, fall back to BCrypt.Net-Next with work factor 12

14. `JwtService` implementing IJwtService:
   - Read JWT config from IConfiguration (Jwt:Key, Jwt:Issuer, Jwt:Audience)
   - Read token lifetimes from SystemSettings table via AuthDbContext (fallback to config values)
   - GenerateAccessToken: Create JWT with claims: sub (userId), email, full_name, branch_id, permissions (as array claim), roles (as array claim). Sign with SymmetricSecurityKey
   - GenerateRefreshToken: Cryptographically random 64-byte token, base64 encoded
   - AccessToken default: 15 minutes. RefreshToken default: 7 days. RememberMe RefreshToken: 30 days

15. `AuthService` implementing IAuthService:
   - LoginAsync: Find user by email (include UserRoles -> Role -> RolePermissions -> Permission), verify password with PasswordHasher, generate access + refresh tokens, store RefreshToken entity with FamilyId, publish UserLoggedInEvent. Return LoginResponse with UserDto
   - RefreshTokenAsync: Find refresh token, verify not expired/revoked, find user, generate new tokens, rotate refresh token (revoke old, create new with same FamilyId). If revoked token is reused, revoke ENTIRE family (security: indicates token theft)
   - LogoutAsync: Revoke the specific refresh token

Use Context7 for Wolverine.HTTP endpoint attributes, FluentValidation AbstractValidator, and EF Core SaveChanges patterns.
