Create architecture tests using NetArchTest.eNhancedEdition to enforce module boundary rules:

**Test Project Setup:**

1. Create `backend/tests/Ganka28.ArchitectureTests/Ganka28.ArchitectureTests.csproj`:
   - Target net10.0
   - Reference: xUnit, NetArchTest.eNhancedEdition (1.4.5), FluentAssertions
   - Reference ALL module projects (to analyze their assemblies)
   - Add to solution

**Module Boundary Tests (ModuleBoundaryTests.cs):**

2. For EACH module (Auth, Audit, Patient, Clinical, Scheduling, Pharmacy, Optical, Billing, Treatment), verify:

   a. `{Module}_Domain_Should_Not_Reference_Other_Modules`:
   - Types in {Module}.Domain namespace should NOT have dependencies on any other module's namespace
   - Exception: references to Shared.Domain are allowed

   b. `{Module}_Domain_Should_Not_Reference_Infrastructure`:
   - Types in {Module}.Domain should NOT reference {Module}.Infrastructure or any Infrastructure namespace
   - Domain must be pure -- no EF Core, no HTTP, no external service references

   c. `{Module}_Application_Should_Not_Reference_Other_Module_Internals`:
   - Types in {Module}.Application should NOT reference other modules' Domain, Application, or Infrastructure
   - Only references to other modules' Contracts namespaces are allowed
   - References to Shared.* are allowed

   d. `{Module}_Contracts_Should_Not_Reference_Module_Internals`:
   - Types in {Module}.Contracts should NOT reference {Module}.Domain, {Module}.Application, or {Module}.Infrastructure
   - Contracts is the public face -- must be self-contained (may reference Shared.Contracts)

   e. `No_Module_Should_Reference_Bootstrapper`:
   - No module type should reference the Bootstrapper assembly

3. Use a data-driven test pattern to avoid repeating test code for each module. Create a test method that takes module name as parameter and runs all boundary checks.

**Dependency Direction Tests (DependencyDirectionTests.cs):**

4. Verify the 4-layer dependency direction is correct:

   a. `Domain_Should_Not_Depend_On_Application`:
   - All types in *.Domain namespaces should NOT reference *.Application namespaces

   b. `Domain_Should_Not_Depend_On_Infrastructure`:
   - All types in *.Domain namespaces should NOT reference *.Infrastructure namespaces

   c. `Application_Should_Not_Depend_On_Infrastructure`:
   - All types in *.Application namespaces should NOT reference *.Infrastructure namespaces (dependency goes the other way: Infrastructure references Application)

   d. `Contracts_Should_Be_Independent`:
   - All types in *.Contracts namespaces should NOT reference *.Domain, *.Application, or *.Infrastructure namespaces
   - Exception: Shared.Contracts references are allowed

**Shared Kernel Tests (SharedKernelTests.cs):**

5. Verify shared kernel usage:

   a. `All_Aggregate_Roots_Should_Have_BranchId`:
   - All types inheriting from AggregateRoot should have a BranchId property (verifies ARC-02 compliance)

   b. `All_Domain_Entities_Should_Have_Private_Setters`:
   - All properties in types within *.Domain.Entities namespaces should NOT have public setters (DDD invariant)

   c. `All_Auditable_Entities_Should_Implement_IAuditable`:
   - Heuristic check: entities in Domain layers that have user-facing data should implement IAuditable

   d. `All_Domain_Events_Should_Implement_IDomainEvent`:
   - All types in *.Domain.Events namespaces should implement IDomainEvent

Use Context7 for NetArchTest.eNhancedEdition assertion patterns and xUnit data-driven test attributes.
