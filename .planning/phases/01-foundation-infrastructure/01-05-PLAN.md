Build the complete login flow and session management:

**API Layer (features/auth/api/auth-api.ts):**

1. Create TanStack Query hooks wrapping the auth API:
   - `loginMutation` -- POST /api/auth/login with LoginRequest, returns LoginResponse
   - `refreshMutation` -- POST /api/auth/refresh with RefreshTokenRequest
   - `logoutMutation` -- POST /api/auth/logout
   - `updateLanguageMutation` -- PUT /api/auth/language
   - `getMeQuery` -- GET /api/auth/me
   - Use openapi-fetch client from shared/lib/api-client.ts

**Auth Hook (features/auth/hooks/useAuth.ts):**

2. `useAuth` custom hook:
   - login(email, password, rememberMe) -- calls loginMutation, on success stores user + accessToken in Zustand authStore, updates i18n language to user's preferredLanguage
   - logout() -- calls logoutMutation, clears authStore, redirects to /login
   - refresh() -- calls refreshMutation, updates tokens in authStore. On failure (refresh token expired), clear auth and redirect to login
   - isAuthenticated -- from authStore
   - user -- from authStore
   - Set up automatic token refresh: schedule refresh before access token expiry (e.g., at 80% of lifetime)

**Session Hook (features/auth/hooks/useSession.ts):**

3. `useSession` custom hook:
   - Track user activity (mousemove, keydown, click events)
   - Timer that counts down from session timeout duration (default 30 min, read from user settings if available)
   - When timer reaches warning threshold (2 minutes before timeout -- Claude's discretion): show warning
   - `showWarning` (boolean), `remainingSeconds` (number), `extendSession()` (resets timer + calls refresh), `isTimedOut` (boolean)
   - On timeout: call logout and redirect to /login

**Login Page (features/auth/components/LoginPage.tsx):**

4. Split layout per user decision:
   - Left side (hidden on mobile): Clinic branding area with gradient background, Ganka28 logo/name, tagline. Use placeholder image/gradient.
   - Right side: Login form centered vertically
   - Responsive: On mobile (<md breakpoint), full-width login form only

**Login Form (features/auth/components/LoginForm.tsx):**

5. Form using React Hook Form + Zod:
   - Fields: Email (input), Password (input with show/hide toggle), Remember Me (checkbox)
   - Validation: Email required + valid format, Password required + min 8 chars
   - Submit: Call useAuth().login()
   - Error handling: Show error message on invalid credentials (from API response), show toast via sonner on network errors
   - Loading state: Disable button, show spinner during login
   - All text uses `useTranslation('auth')` for bilingual support

**Session Warning Modal (features/auth/components/SessionWarningModal.tsx):**

6. shadcn/ui Dialog that appears when session is about to expire:
   - Shows countdown timer: "Phien dang nhap se het han trong X phut Y giay" / "Session expires in X min Y sec"
   - Two buttons: "Gia han" / "Extend" (calls extendSession), "Dang xuat" / "Logout" (calls logout)
   - Modal is non-dismissible (no close on backdrop click)
   - All text bilingual via i18n

**Login Route (app/routes/login.tsx):**

7. Update the login route:
   - Render LoginPage component
   - If already authenticated, redirect to /dashboard
   - Handle redirect parameter: after login, redirect to the originally requested page

**Language Toggle Integration:**

8. Update LanguageToggle component to also call the backend API to persist language preference:
   - On toggle: call i18n.changeLanguage(newLang) immediately (instant UI update)
   - Then call updateLanguageMutation to persist to backend (background, no blocking)

Use Context7 for React Hook Form + Zod integration, TanStack Query mutations, and shadcn/ui Dialog component.
