## Decisions Made
- **DataTable receives table instance, not raw data:** Consumers create and configure useReactTable themselves, DataTable only handles rendering. This preserves full control over sorting, pagination, expansion, and manual pagination modes.
- **Separate edit/create Controller for fullName:** TypeScript requires separate Controller elements for different form Control types (CreateFormValues vs EditFormValues) rather than conditional control prop.
- **AuditLogFilters uses Field+FieldLabel without Controller:** Filter bars are controlled components (state via props, not React Hook Form), so they get consistent styling via Field+FieldLabel without needing Controller or FieldError.
- **headerStyle prop on DataTable:** Added to support AuditLogTable's custom column widths (expander column at 40px) without hardcoding in DataTable.
