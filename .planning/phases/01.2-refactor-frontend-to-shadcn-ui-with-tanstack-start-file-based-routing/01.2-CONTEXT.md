<domain>
## Phase Boundary

Refactor the existing frontend to use shadcn/ui comprehensively. Replace the current dashboard layout with shadcn's `dashboard-01` block, replace the login page with shadcn's `login-04` block, establish a wrapper-based component maintenance strategy for safe shadcn/ui upgrades, and rebuild all existing feature components (tables, forms, modals) using shadcn/ui patterns. File-based routing with TanStack Start is already in place and stays unchanged.

<decisions>
## Implementation Decisions

### Component maintenance strategy
- Wrapper component pattern: every shadcn/ui primitive gets a project wrapper in `shared/components/` (e.g., `shared/components/Button.tsx` wraps `shared/components/ui/button.tsx`)
- `ui/` folder is treated as internal/private — never imported directly by feature or page code
- Strict import rule: ALL components in the codebase import from wrappers, never from `ui/` directly. Enforce by convention or lint rule.
- Upgrade path: Claude's discretion on safest approach

### Dashboard layout (dashboard-01)
- Replace AppShell entirely with dashboard-01's layout (sidebar + header pattern)
- Existing AppSidebar and TopBar get replaced by dashboard-01's sidebar and header
- Dashboard-01 header replaces current TopBar — adapt language toggle and user menu into it
- Navigation: expand with placeholder items for future phases (Patients, Appointments, etc.) shown as disabled/coming-soon
- Page content: use dashboard-01 layout structure, but page content depends on current implementation (no chart components needed now)

### Login page (login-04)
- Replace current LoginPage entirely with login-04 block
- Use login-04's image/illustration panel instead of current gradient + logo panel — adapt Ganka28 branding into it
- Remove all social login buttons (GitHub, Google, etc.) — clinic system uses username/password only
- Keep username + password fields (not email) — matches existing backend auth contract
- Language toggle stays in top-right corner of the form panel

### Migration scope
- All existing feature components (UserTable, RoleTable, AuditLogTable, PermissionMatrix, UserManagementPage, RoleManagementPage, AuditLogPage) rebuilt to use shadcn/ui patterns
- Tables rebuilt using shadcn Data Table pattern (TanStack Table + shadcn)
- Forms rebuilt with full shadcn Form pattern (React Hook Form + Zod + shadcn Form component) — consistent pattern across all forms
- SessionWarningModal rebuilt as shadcn AlertDialog (non-dismissible by default, fits session warning use case)
- File-based routing: no changes needed — current TanStack Start routing stays as-is

### Claude's Discretion
- Exact upgrade strategy for shadcn/ui (re-run CLI vs keep ui/ untouched)
- Chart component inclusion timing
- Exact placeholder nav item list based on roadmap phases
- Dashboard-01 image/branding panel design adaptation
- Loading skeleton and error state patterns

</decisions>
