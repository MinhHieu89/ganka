<objective>
Fix the PharmacyOptical auto-advance-to-Done bug reported in UAT. Currently, visits at PharmacyOptical (stage 7) are treated as completed and immediately sorted into the Done column. The fix adds a new `Done = 8` enum value so that PharmacyOptical is a real working stage and Done requires explicit advancement.

Purpose: Users must be able to work with visits in the PharmacyOptical stage without them auto-disappearing into Done.
Output: Backend enum + logic updated, frontend kanban and table view updated, all tests passing.
</objective>
