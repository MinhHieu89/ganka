    **Stage2RefractionView.tsx:**
    Complete Stage 2 detail view per spec. Uses StageDetailShell wrapper.

    Form layout:
    - Two-column: left = OD (labeled `MP (Mắt phải)`), right = OS (labeled `MT (Mắt trái)`)
    - Each column: SPH, CYL, AXIS, ADD, PD. Second section: uncorrected VA, corrected VA, IOP, axial length, method selector.
    - Three tabs at top: `Thường quy` / `Máy đo tự động` / `Liệt điều tiết` (maps to existing refraction types)

    Field validation per spec:
    - SPH (OD) and SPH (OS) are the ONLY required fields
    - Empty SPH: red border + red background tint (`border-red-500 bg-red-50`)
    - Filled SPH: green border + green background tint (`border-green-500 bg-green-50`)
    - Optional fields: no highlight ever, regardless of state

    Validation states (shown via ValidationMessage component):
    - Both SPH empty: error — `"Chưa thể chuyển — cần nhập SPH cho mắt phải và mắt trái."`
    - One SPH filled: warning — `"Mắt phải đã nhập. Cần nhập SPH cho mắt trái để chuyển tiếp."` (or vice versa)
    - Both SPH filled: success — `"Sẵn sàng chuyển — bác sĩ sẽ nhận lượt khám ngay."`

    Bottom bar (via StageBottomBar):
    - `Lưu nháp` — always enabled, calls existing save refraction mutation
    - `Bỏ qua bước này` — red-bordered secondary button (`variant="outline" className="border-red-500 text-red-600"`), opens SkipStageModal
    - `Hoàn tất đo, chuyển bác sĩ ›` — primary black button, disabled until both SPH filled. Calls useAdvanceStage to advance from RefractionVA(1) to DoctorExam(2).

    **SkipStageModal.tsx:**
    Reusable skip modal (will be used only for Stage 2 in v1 but designed reusable).
    - Title: `"Bỏ qua đo khúc xạ / thị lực?"`
    - Preset reason chips (single-select, mandatory):
      - `Tái khám, đã có kết quả cũ` (FollowUpExisting=0)
      - `Bệnh nhân từ chối đo` (PatientRefused=1)
      - `Khám tổng quát, không liên quan khúc xạ` (UnrelatedExam=2)
      - `Khác` (Other=3)
    - Optional free-text note: max 200 chars with live character counter
    - Confirm button disabled until a chip is selected
    - Footer: `Hủy` (cancel) / `Xác nhận bỏ qua` (destructive red confirm button)
    - On confirm: calls useSkipRefraction mutation

    **RefractionSkipBanner.tsx:**
    Yellow warning banner shown when refraction is skipped.
    - Background: `bg-amber-50 border border-amber-300`
    - Text: `"Đo khúc xạ đã bỏ qua — Lý do: [reason] · [actor] · [time]"`
    - `Hoàn tác` (Undo) button on right side. Calls useUndoRefractionSkip mutation.
    - Undo only available while visit is still at RefractionVA or DoctorExam stage.

    Post-skip state in Stage2RefractionView:
    - Form fields get `opacity-50 pointer-events-none` classes
    - Stage pill changes to amber `Đã bỏ qua`
    - Bottom bar: `Lưu nháp` + `Chuyển bác sĩ ›` (shortened label since nothing to complete)

    **Route file (visit.$visitId.stage.tsx):**
    Create a route that loads the appropriate stage view based on the visit's current stage. For now, only Stage 2 is implemented — other stages will show a placeholder. The route pattern: `/clinical/visit/:visitId/stage`
    - Fetch visit by ID
    - Switch on currentStage to render the correct stage view component
    - Stage 1 (Reception): render existing forward-only view (or simple placeholder)
    - Stage 2 (RefractionVA): render Stage2RefractionView
    - All other stages: render placeholder with stage name
