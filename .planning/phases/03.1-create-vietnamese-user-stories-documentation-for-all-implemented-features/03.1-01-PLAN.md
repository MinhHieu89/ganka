**Requirements to cover as user stories:**

1. **AUTH-01** (Login with JWT): US-AUTH-001 - Staff login with email/password. Include: login form fields (Email, Mat khau, Ghi nho dang nhap), JWT token with role-based claims, redirect to Dashboard. Error: wrong credentials, disabled account. Technical notes: POST /api/auth/login, HTTP-only cookie for refresh token (per Phase 02.1 decision), bcrypt validation.

2. **AUTH-02** (Role support): US-AUTH-002 - System supports 8 roles: Admin (Quan tri vien), Doctor (Bac si), Technician (Ky thuat vien), Nurse (Dieu duong), Cashier (Thu ngan), OpticalStaff (Nhan vien kinh mat), Manager (Quan ly), Accountant (Ke toan). Include: role assignment by admin, preset permission templates per role.

3. **AUTH-03** (Granular permissions): US-AUTH-003 - Admin configures CRUD permissions per role per entity/action. Include: permission matrix UI, role creation with custom permissions, permission inheritance from preset templates. Edge case: removing a permission mid-session.

4. **AUTH-04** (Session persistence): US-AUTH-004 - Session persists across browser refresh via HTTP-only cookie refresh token. 30-min inactivity timeout with 2-min warning dialog (AlertDialog, non-dismissible). Activity throttled to 30s intervals. Include: "Remember me" for longer session. Edge cases: multiple tabs, token refresh flow. Technical: silent refresh on page load (Phase 02.1), credentials:include on all API calls.

5. **AUTH-05** (Login/access logging): US-AUTH-005 - All login attempts logged (success and failure). All record access and data views logged. Cross-reference to 05-nhat-ky (audit) stories.

**Also include admin workflow stories:**
- US-AUTH-006: Admin creates new user (name, email, role assignment)

- US-AUTH-008: Admin creates and configures roles with permission matrix

**Requirements to cover as user stories:**

1. **PAT-01** (Register medical patient): US-PAT-001 - Staff registers medical patient with: name, phone, DOB, gender. System auto-generates GK-YYYY-NNNN patient code (application-level, MAX+1 per year). Include: form fields, validation rules (name 3-50 chars), RowVersion concurrency. Error: duplicate phone, missing required fields. Technical: POST /api/patients, Vietnamese_CI_AI collation for name search.

2. **PAT-02** (Walk-in pharmacy customer): US-PAT-002 - Staff registers walk-in customer with name + phone only (lightweight record). Include: PatientType distinction, no full medical profile required. Edge case: converting walk-in to full medical patient.

3. **PAT-03** (Configurable mandatory fields): US-PAT-003 - System supports configurable mandatory fields (Address, CCCD) that become required for referrals, legal export, or So Y te reporting. Include: PatientFieldValidationResult records in Domain.Services, referral context as strictest downstream. Technical: validation endpoint.

4. **PAT-04** (Patient search): US-PAT-004 - Staff searches patients by name, phone, or patient ID. Results in <=3 seconds. Include: GlobalSearch component (Ctrl+K shortcut), Command+Popover pattern, recent patients on focus, deferred API search. Vietnamese_CI_AI collation for accent-insensitive search. Edge case: no results.

5. **PAT-05** (Allergy management): US-PAT-005 - Staff records structured allergy list per patient. Include: allergy combobox (Button trigger with div wrapper, free-text support), severity levels, Vietnamese allergy categories (categoryKeyMap for i18n). AllergyAlert component with full banner + compact tooltip modes. Cross-reference: "Xem them: US-CLN-xxx" for clinical workflow allergy display. Technical: AllergyCatalogSeeder, allergy autocomplete stores English canonical key.

**Also include additional workflow stories:**
- US-PAT-006: Staff views patient profile (tabs: overview, appointments, clinical history)
- US-PAT-007: Staff edits patient information (inline edit with optimistic concurrency)
- US-PAT-008: Staff uploads patient photo (native fetch with FormData, not openapi-fetch)

**Story format:** Same as Task 1 -- use the standard template with proper Vietnamese diacritics throughout. Reference `frontend/public/locales/vi/patient.json` for Vietnamese UI labels.
