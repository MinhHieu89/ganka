**Requirements to cover as user stories:**

**Clinical Workflow (CLN-01..04):**

1. **CLN-01** (Visit record, immutable after sign-off): US-CLN-001 - Doctor creates electronic visit record (benh an/luot kham) linked to patient and doctor. After sign-off, record becomes immutable. Include: SignOff uses AlertDialog (non-dismissible) for confirmation. VisitSection as reusable collapsible card wrapper with defaultOpen prop and headerExtra slot.

2. **CLN-02** (Amendment records): US-CLN-002 - Corrections to signed records create amendment records preserving original. Include: VisitAmendment.FieldChangesJson stores JSON array of FieldChange records for field-level diff. Capture signed-state snapshot at amendment initiation (not before/after diff). Amendment requires reason text.

3. **CLN-03** (Workflow status tracking): US-CLN-003 - Staff tracks visit workflow through 8 stages. 5 Kanban columns: Tiep nhan (Stage 0), Kham nghiem (Stage 1), Bac si (Stages 2,3,4), Xu ly (Stages 5,6), Hoan tat (Stage 7). Include: @dnd-kit drag-and-drop (PointerSensor distance:8 + TouchSensor delay:200). Wait time badge: destructive at 60+ min, secondary at 30+, outline below.

4. **CLN-04** (Dashboard with active patients): US-CLN-004 - Dashboard shows all active patients and their current workflow stage in real-time. Include: Kanban board with drag-to-advance, patient card details, stage transitions.

**Refraction & Examination (REF-01..03):**

5. **REF-01** (Refraction data): US-CLN-005 - Technician or doctor records refraction data: SPH, CYL, AXIS, ADD, PD per eye (OD/OS). Include: RefractionForm with debounced auto-save on blur (500ms). Decimal fields use precision(5,2). RefractionDto uses 'type' (read path) while updateRefraction sends 'refractionType' (write path).

6. **REF-02** (VA, IOP, Axial Length): Combine with US-CLN-005 or create US-CLN-006. Record VA (with/without correction), IOP (with method Select - uses empty string for no-selection state, and time), Axial Length per eye. IOP method stored as enum.

7. **REF-03** (Refraction types): Part of US-CLN-005/006. System supports manifest, autorefraction, and cycloplegic refraction types. Each type stored as separate refraction record.

**Diagnosis (DX-01..02):**

8. **DX-01** (ICD-10 search): US-CLN-007 - Doctor searches ICD-10 codes in Vietnamese and English with favorites pinned. Include: Icd10Combobox uses Button trigger (not Input) with div wrapper. SearchIcd10Codes uses Contains for bilingual search with favorites pinned via OrderByDescending. DoctorIcd10Favorite as per-doctor junction table (not global IsFavorite field).

9. **DX-02** (ICD-10 laterality): US-CLN-008 - System enforces laterality selection (OD/OS/OU) for ophthalmology codes. Include: inline laterality selector in Icd10Combobox. OU creates two diagnosis records with .1/.2 code suffixes per ICD-10 convention. Non-laterality codes default to laterality 0 (OD).

**Also include workflow stories:**
- US-CLN-009: Staff views allergy alerts during clinical workflow (cross-ref: "Xem them: US-PAT-005")
- US-CLN-010: Doctor views visit summary before sign-off

**Story format:** Use the standard template from RESEARCH.md with proper Vietnamese diacritics throughout. Reference `frontend/public/locales/vi/clinical.json` for Vietnamese UI labels.

This is the largest file -- it covers 9 requirements. Ensure each requirement gets adequate coverage without the file becoming unwieldy. Group related refraction stories (REF-01, REF-02, REF-03) into 1-2 stories since they describe the same form/workflow.
