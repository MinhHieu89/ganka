**Requirements to cover as user stories:**

**UI Requirements:**

Include: i18next integration, vi/*.json and en/*.json translation files, Vietnamese as default language. All Vietnamese text uses proper diacritics (per Phase 01.2-07 decision).

2. **UI-02** (Language switching): US-SYS-002 - Staff can switch language preference per user session. Include: LanguageToggle component, language preference persisted via UpdateLanguage API endpoint (per user, not per browser).

**Architecture Requirements (frame as system-level stories from Admin/Manager perspective):**

3. **ARC-01** (ACL adapter pattern): US-SYS-003 - "La mot quan tri vien, toi muon he thong su dung kien truc mo-dun de tich hop voi cac dich vu ben ngoai, de co the thay doi nha cung cap ma khong anh huong den nghiep vu." Include: domain ports + infrastructure adapters pattern. Technical notes about ACL pattern implementation.

4. **ARC-02** (BranchId multi-branch): US-SYS-004 - "La mot quan tri vien, toi muon he thong ho tro da chi nhanh tu kien truc, de co the mo rong khi phong kham phat trien." Include: BranchId on all aggregate roots, EF Core global query filters. Note: single-branch for v1, architecture ready for expansion.

5. **ARC-03** (Template engine extensibility): US-SYS-005 - "La mot quan tri vien, toi muon he thong ho tro them mau benh moi (disease templates) ma khong can thay doi ma nguon, de co the mo rong cho cac benh khac ngoai Dry Eye." Include: config/plugin-driven templates, extensible architecture.

6. **ARC-04** (Azure SQL daily backup): US-SYS-006 - "La mot quan tri vien, toi muon he thong tu dong sao luu du lieu hang ngay, de dam bao khong mat du lieu khi co su co." Include: Azure SQL automatic daily backup, point-in-time recovery (35 days retention).

7. **ARC-05** (Azure Blob Storage): US-SYS-007 - "La mot quan tri vien, toi muon he thong luu tru hinh anh y khoa an toan voi kha nang khoi phuc, de bao ve du lieu hinh anh cua benh nhan." Include: Azure Blob Storage with soft delete and versioning, StorageBlobInfo (renamed to avoid Azure SDK collision).

8. **ARC-06** (Full data export): US-SYS-008 - "La mot quan tri vien, toi muon co kha nang xuat toan bo du lieu cua he thong, de dam bao quyen so huu du lieu va khong bi rang buoc voi nha cung cap." Include: data export capability, no vendor lock-in guarantee.

**Story format:** Use the standard template from RESEARCH.md with proper Vietnamese diacritics throughout. Reference `frontend/public/locales/vi/common.json` for Vietnamese UI labels.

**Important note on ARC-* stories:** These are internal/infrastructure requirements. Frame them from the Admin/Manager perspective to make them meaningful as user stories. Acceptance criteria should be more technical since the audience is dev/QA. The "benefit" in each story should focus on business value (data safety, extensibility, no vendor lock-in) rather than technical elegance.
