    1. StocktakingSession : AggregateRoot, IAuditable in Optical.Domain.Entities:
       Properties:
       - string Name (e.g., "Monthly Stocktake - March 2026")
       - StocktakingStatus Status (InProgress, Completed, Cancelled)
       - Guid StartedById (staff who started)
       - DateTime? CompletedAt
       - string? Notes

       Child collection:
       - private readonly List<StocktakingItem> _items = new()
       - public IReadOnlyList<StocktakingItem> Items => _items.AsReadOnly()

       Methods:
       - static StocktakingSession Create(name, startedById, BranchId) -- Status = InProgress
       - StocktakingItem RecordItem(string barcode, int physicalCount, int systemCount) -- UPSERT pattern: if barcode already in _items, update its physicalCount; otherwise add new StocktakingItem. This prevents duplicate scans (per RESEARCH.md pitfall 5). Returns the created/updated item.
       - void Complete() -- sets Status=Completed, CompletedAt=UtcNow. Throws if already Completed/Cancelled.
       - void Cancel() -- sets Status=Cancelled. Throws if already Completed.
       - int TotalItemsScanned => _items.Count
       - int DiscrepancyCount => _items.Count(i => i.PhysicalCount != i.SystemCount)

    2. StocktakingItem : Entity in Optical.Domain.Entities:
       Properties:
       - Guid StocktakingSessionId (FK)
       - string Barcode (EAN-13 barcode scanned)
       - Guid? FrameId (resolved from barcode, nullable if barcode not found in system)
       - string? FrameName (human-readable: "Rayban RB5154 Black 52-18-140")
       - int PhysicalCount (entered by staff)
       - int SystemCount (from inventory at time of scan)
       - int Discrepancy => PhysicalCount - SystemCount

       Methods:
       - static StocktakingItem Create(sessionId, barcode, frameId, frameName, physicalCount, systemCount)
       - void UpdatePhysicalCount(int newCount) -- for correction during stocktaking
