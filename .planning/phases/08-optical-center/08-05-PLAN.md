    - private readonly List<GlassesOrderItem> _items = new()

    Static allowed transitions dictionary (as shown in RESEARCH.md):
    - Ordered -> [Processing]
    - Processing -> [Received]
    - Received -> [Ready]
    - Ready -> [Delivered]

    Methods:
    - static GlassesOrder Create(patientId, patientName, visitId, opticalPrescriptionId, processingType, estimatedDeliveryDate, totalPrice, comboPackageId, notes, BranchId)
    - void TransitionTo(GlassesOrderStatus newStatus) -- validates against AllowedTransitions, throws InvalidOperationException for invalid transitions. Sets DeliveredAt=DateTime.UtcNow when transitioning to Delivered. Calls SetUpdatedAt() and AddDomainEvent(new GlassesOrderStatusChangedEvent(...))
    - void ConfirmPayment() -- sets IsPaymentConfirmed = true, calls SetUpdatedAt()
    - void AddItem(GlassesOrderItem item) -- adds to _items collection
    - bool IsUnderWarranty => DeliveredAt.HasValue && DeliveredAt.Value.AddMonths(12) > DateTime.UtcNow
    - bool IsOverdue => EstimatedDeliveryDate.HasValue && Status != GlassesOrderStatus.Delivered && DateTime.UtcNow > EstimatedDeliveryDate.Value

    IMPORTANT: TransitionTo for Ordered->Processing must NOT check payment here -- that's the handler's job (OPT-04). The entity only manages status transitions. Payment enforcement is in the application layer.

    Private parameterless constructor for EF Core.
