  <action>
NumberInput.onChange signature is `(value: number) => void`. The `value` parameter IS the number directly, not an event object. Fix all three files:

**1. ComboPackageForm.tsx — line 360-362 (originalTotalPrice field)**

Replace:
```tsx
onChange={(e) =>
  field.onChange(e.target.value === "" ? undefined : Number(e.target.value))
}
```

With:
```tsx
onChange={(value) => field.onChange(value)}
```

The NumberInput component internally handles empty-string-to-NaN conversion and only calls onChange with valid numbers. The `value` passed is always a `number`. The `undefined` fallback is unnecessary because NumberInput only fires onChange when a valid number is parsed.

**2. WarrantyClaimForm.tsx — line 320-323 (discountAmount field)**

Replace:
```tsx
onChange={(e) => {
  const val = e.target.value
  field.onChange(val === "" ? null : Number(val))
}}
```

With:
```tsx
onChange={(value) => field.onChange(value)}
```

Same reasoning — NumberInput only calls onChange with a parsed number.

**3. StocktakingScanner.tsx — line 147 (physicalCount field)**

Replace:
```tsx
onChange={(e) => setPhysicalCount(parseInt(e.target.value, 10) || 0)}
```

With:
```tsx
onChange={(value) => setPhysicalCount(value || 0)}
```

The `value` is already a number from NumberInput, so parseInt is unnecessary. The `|| 0` fallback handles NaN/0 cases.
