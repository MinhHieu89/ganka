    Add endpoints following the existing Minimal API pattern in each Presentation project.

    **Scheduling endpoints** (add to existing SchedulingEndpoints.cs route group):

    ```
    POST /api/scheduling/appointments/check-in
      Body: { appointmentId: Guid }
      Auth: Scheduling.Update
      Dispatches: CheckInAppointmentCommand (UserId from HttpContext)
      Returns: 200 { visitId: Guid }

    POST /api/scheduling/appointments/guest
      Body: { guestName, guestPhone, guestReason?, doctorId?, doctorName?, startTime, source }
      Auth: Scheduling.Create
      Dispatches: BookGuestAppointmentCommand
      Returns: 201 { appointmentId: Guid }

    POST /api/scheduling/appointments/{id}/no-show
      Body: { notes?: string }
      Auth: Scheduling.Update
      Dispatches: MarkAppointmentNoShowCommand (UserId from HttpContext)
      Returns: 200

    GET /api/scheduling/slots?date={date}&doctorId={doctorId?}
      Auth: Scheduling.View
      Dispatches: GetAvailableSlotsQuery
      Returns: 200 AvailableSlotDto[]

    GET /api/scheduling/receptionist/dashboard?status={filter}&search={search}&page={page}&pageSize={pageSize}
      Auth: Scheduling.View
      Dispatches: GetReceptionistDashboardQuery
      Returns: 200 { items: ReceptionistDashboardRowDto[], totalCount: int }

    GET /api/scheduling/receptionist/kpi
      Auth: Scheduling.View
      Dispatches: GetReceptionistKpiStatsQuery
      Returns: 200 ReceptionistKpiDto
    ```

    **Clinical endpoints** (add to existing ClinicalEndpoints.cs):

    ```
    POST /api/clinical/visits/walk-in
      Body: { patientId, doctorId, reason? }
      Auth: Clinical.Create
      Dispatches: CreateWalkInVisitCommand
      Returns: 201 { visitId: Guid }

    POST /api/clinical/visits/{id}/cancel-with-reason
      Body: { reason: string }
      Auth: Clinical.Update (or Clinical.Create since receptionist has it)
      Dispatches: CancelVisitWithReasonCommand (UserId from HttpContext)
      Returns: 200
    ```

    **Patient endpoints** (add to existing PatientEndpoints.cs):

    ```
    POST /api/patients/intake
      Body: RegisterPatientFromIntakeCommand fields
      Auth: Patient.Create
      Dispatches: RegisterPatientFromIntakeCommand
      Returns: 201 { patientId: Guid }

    PUT /api/patients/{id}/intake
      Body: UpdatePatientFromIntakeCommand fields
      Auth: Patient.Update
      Dispatches: UpdatePatientFromIntakeCommand
      Returns: 200
    ```

    Each endpoint:
    - Uses `RequireAuthorization(policy)` matching the permission constant from Shared.Domain.Permissions
    - Extracts UserId from `HttpContext.User` using existing `TryGetUserId()` extension
    - Returns proper status codes (200 for queries/mutations, 201 for creates)
    - Uses `.ToHttpResult()` for Result<T> handling
