## Spacing Scale

Declared values (multiples of 4, mapped to Tailwind):

| Token | Value | Tailwind | Usage |
|-------|-------|----------|-------|
| xs | 4px | `p-1` / `gap-1` | Icon gaps, inline badge padding |
| sm | 8px | `p-2` / `gap-2` | Compact element spacing, table cell padding |
| md | 16px | `p-4` / `gap-4` | Default element spacing, card padding |
| lg | 24px | `p-6` / `gap-6` | Section padding, form section gaps |
| xl | 32px | `p-8` / `gap-8` | Layout gaps between major sections |
| 2xl | 48px | `p-12` / `gap-12` | Page-level vertical spacing |

Exceptions:
- KPI cards: 16px internal padding, 16px gap between cards
- Table rows: 12px vertical padding (`py-3`) for comfortable scanning
- Dialog popups: 24px padding all around
- Form sections (collapsible): 16px padding inside, 24px gap between sections
- Action bar: 8px gap between buttons, 16px gap between button group and search

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 (20px) | Table cells, form field values, dialog body text |
| Label | 14px | 500 (medium) | 1.4 (20px) | Form labels, table column headers, KPI card labels |
| Heading | 20px | 600 (semibold) | 1.2 (24px) | Page titles ("Dashboard"), dialog headers, section headers |
| Display | 28px | 700 (bold) | 1.1 (32px) | KPI card numbers only |

Additional typography rules:
- Patient name in table: 14px, weight 500 (medium) -- stands out from regular body
- Patient code (GK-YYYY-NNNN): 14px, `font-mono` (JetBrains Mono), `text-muted-foreground`
- Appointment time in table: 14px, weight 400
- "Chua ro" (unknown reason): 14px, italic, `text-muted-foreground`
- Not-arrived reason text: 14px, italic
- Slot availability header: 14px, weight 500
- Confirmation bar summary: 14px, weight 500

### Badge Variant Extensions

The existing `Badge` component has `default`, `secondary`, `destructive`, `outline` variants. This phase requires custom status/source badge styling via `className` overrides (Tailwind utility classes using the CSS variables above), NOT new CVA variants. This keeps the badge component generic.

Usage pattern:
```tsx
<Badge className="border-transparent bg-[var(--status-not-arrived-bg)] text-[var(--status-not-arrived-text)]">
  Chua den
</Badge>
```

## Interaction Contracts

### Dashboard Polling

| Data | Interval | Method |
|------|----------|--------|
| KPI card counts | 30 seconds | TanStack Query `refetchInterval: 30000` |
| Patient table rows | 15 seconds | TanStack Query `refetchInterval: 15000` |
| Manual refresh | On click | `queryClient.invalidateQueries()` for all dashboard queries |

### Table Behaviors

| Behavior | Spec |
|----------|------|
| Default sort | "Gio hen" ascending; walk-ins (no appointment time) sort to bottom by arrival time |
| Sortable columns | Ho ten, Nam sinh, Gio hen, Nguon, Trang thai |
| Not-arrived row styling | `bg-secondary` (stone-50 tint) to visually separate from arrived patients |
| Row click on name | Navigate to Intake form (view mode) |
| Pagination default | 10 rows per page; options: 10, 20, 50 |
| Column widths | STT: 48px, Ho ten: flex-1 (min 160px), Nam sinh: 80px, Gio hen: 80px, Nguon: 80px, Ly do kham: flex-1 (min 120px), Trang thai: 100px, Thao tac: 100px |

### Check-in Button (Not Arrived status only)

| Property | Value |
|----------|-------|
| Style | Outline, border and text `#534AB7` |
| Label | "Check-in" |
| Position | Thao tac column, replaces "..." menu |
| Hover | `bg-[#EEEDFE]` |

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Focus management | Dialog trap focus; return focus to trigger on close |
| Keyboard nav | DropdownMenu arrow keys; Dialog Escape to close; Table sortable columns via Enter |
| ARIA labels | All icon-only buttons must have `aria-label`; status badges use `role="status"` |

| Screen reader | KPI cards: `aria-live="polite"` for count updates; Toast announcements via Sonner |
