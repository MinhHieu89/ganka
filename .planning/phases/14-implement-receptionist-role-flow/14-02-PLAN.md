    **CheckInAppointment handler:**
    - Command: `CheckInAppointmentCommand(Guid AppointmentId, Guid UserId)`
    - Load appointment, validate status==Confirmed
    - Call `appointment.CheckIn()`
    - Load patient data (if PatientId not null) to get HasAllergies and DoctorName
    - Create Visit via `Visit.Create(patientId, patientName, doctorId, doctorName, branchId, hasAllergies, appointmentId)` with `source: VisitSource.Appointment`
    - If D-08: auto-advance visit to PreExam stage -- but this is for "Luu & Chuyen tien kham" on intake form, NOT for standard check-in. Standard check-in creates visit at Reception stage.
    - Save, return `Result<Guid>` (visitId)

    **BookGuestAppointment handler (per D-11):**
    - Command: `BookGuestAppointmentCommand(string GuestName, string GuestPhone, string? GuestReason, Guid? DoctorId, string? DoctorName, DateTime StartTime, AppointmentSource Source)`
    - Validate: GuestName + GuestPhone required
    - EndTime = StartTime + 30 minutes (D-10)
    - If DoctorId provided: check no overlap for that doctor (D-12)
    - If DoctorId null: skip overlap check (D-12 "BS nao trong" allows freely)
    - Create via `Appointment.CreateGuest(...)`, persist, return Id

    **MarkAppointmentNoShow handler:**
    - Command: `MarkAppointmentNoShowCommand(Guid AppointmentId, Guid UserId, string? Notes)`
    - Load, validate, call `appointment.MarkNoShow(userId, notes)`, save

    **GetAvailableSlots handler (per D-10):**
    - Query: `GetAvailableSlotsQuery(DateTime Date, Guid? DoctorId)`
    - Load ClinicSchedule for that day of week to get open/close hours
    - Generate 30-min slots between open and close
    - For each slot, count booked appointments (if DoctorId provided, filter by doctor)
    - Return list of AvailableSlotDto with IsAvailable flag

    **GetReceptionistDashboard handler:**
    - Query: `GetReceptionistDashboardQuery(string? StatusFilter, string? Search, int Page, int PageSize)`
    - Query today's appointments (Scheduling module) + today's visits (needs cross-module query)
    - For cross-module: use Wolverine bus to send `GetTodayVisitsQuery` to Clinical module, or use a read-only SQL query joining across schemas. Per Research recommendation: use two separate queries and join in-memory.
    - Map each row to 4 statuses per the mapping table:
      - Not Arrived: Appointment.Status == Confirmed, no linked Visit
      - Waiting: Visit.CurrentStage == Reception
      - Examining: Visit.CurrentStage >= PreExam AND Visit.Status not Signed/Cancelled
      - Completed: Visit.Status == Signed OR Visit.CurrentStage == Done
    - Also include walk-in visits (no appointment) in Waiting/Examining/Completed
    - Default sort: AppointmentTime ascending; walk-ins sort to bottom by CreatedAt
    - Apply filters and search (by PatientName or PatientCode)
    - Return paginated results

    **GetReceptionistKpiStats handler:**
    - Query: `GetReceptionistKpiStatsQuery()` (no params, always today)
    - Same data source as dashboard, but just return counts per status
    - Return ReceptionistKpiDto

    **Tests:** Write failing tests first for each handler. Use moq for repository interfaces. Follow existing test patterns in Scheduling.Unit.Tests.
