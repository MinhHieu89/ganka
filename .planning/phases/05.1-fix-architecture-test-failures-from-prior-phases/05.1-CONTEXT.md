# Phase 05.1: Fix Architecture Test Failures from Prior Phases - Context

**Gathered:** 2026-03-06
**Status:** Ready for planning

<domain>
## Phase Boundary

Fix 5 failing NetArchTest rules across Patient and Clinical modules, restoring clean architecture boundaries that were consciously shortcutted during rapid phase execution. This phase fixes code to comply with existing architecture tests — no new features, no test relaxation.

Failures stem from 4 root causes:
1. Patient.Contracts references Patient.Domain.Enums (2 test failures)
2. Clinical.Application references Shared.Infrastructure ReferenceDbContext (1 test failure)
3. FieldChange record in Entities namespace triggers private-setter test (1 test failure)
4. Patient.Presentation uses PatientFieldValidationResult from Patient.Domain.Services (1 test failure)

</domain>

<decisions>
## Implementation Decisions

### Enum Ownership Strategy
- Duplicate enums in Contracts: each module defines its own Contracts-layer enums (e.g., `Patient.Contracts.Enums.Gender`) parallel to Domain enums (e.g., `Patient.Domain.Enums.Gender`)
- Explicit mapper methods: static `ToContractEnum()` / `ToDomainEnum()` extension methods for type-safe conversion at boundaries — catches mismatches at compile time if enums diverge
- Apply pattern to ALL modules now, not just Patient — ensures consistency across the codebase
- Pharmacy module already uses int normalization in Contracts (compliant), but verify and align if needed

### Reference Data Access (ICD-10)
- Define `IReferenceDataRepository` interface in `Shared.Application` — Application layer owns repository interfaces per established pattern
- Implementation in `Shared.Infrastructure` wraps existing `ReferenceDbContext`
- Clinical.Application depends on `IReferenceDataRepository` (interface) instead of `ReferenceDbContext` (concrete)
- Remove `Shared.Infrastructure` project reference from `Clinical.Application.csproj`
- Register in `Shared.Infrastructure.IoC.cs` DI

### FieldChange Record Fix
- Move `FieldChange` from `Clinical.Domain.Entities` to `Clinical.Domain.ValueObjects` namespace
- FieldChange is semantically a value object (immutable data carrier for amendment field-level diffs) — the namespace move is architecturally correct
- Test scans `*.Domain.Entities` — moving it out fixes the violation without changing the type or test

### PatientFieldValidationResult Fix
- Move `PatientFieldValidationResult` out of `Patient.Domain.Services` to `Patient.Contracts` (it's a DTO-like result type used across boundaries)
- Remove `using Patient.Domain.Services` from `Patient.Presentation.PatientApiEndpoints`
- Clean up any other unused `using Patient.Domain.Enums` directives in Presentation layer

### Claude's Discretion
- Exact mapper method signatures and placement (extension methods in Contracts or Application)
- Whether to create a shared `EnumMapper<TDomain, TContract>` base or keep per-enum mappers
- Additional cleanup of unused imports discovered during fixes
- Test run verification and any cascade fixes

</decisions>
