## Decisions Made
- Presentation layer owns all cookie logic -- handlers remain unaware of HTTP transport
- Added RememberMe to RefreshTokenResponse to propagate flag from domain to endpoint for cookie Max-Age
- Used LocalDB with per-test-run database names instead of Testcontainers (Docker Desktop not running)

- Registered concrete PasswordHasher in test DI because AuthDataSeeder resolves concrete type (not interface)

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] AuthDataSeeder resolves concrete PasswordHasher type**
- **Found during:** Task 1 (RED phase, CustomWebApplicationFactory setup)
- **Issue:** AuthDataSeeder calls `sp.GetRequiredService<PasswordHasher>()` (concrete class), but DI only registers `IPasswordHasher`
- **Fix:** Added `services.AddSingleton<PasswordHasher>()` in test factory
- **Files modified:** CustomWebApplicationFactory.cs
- **Verification:** Application starts successfully, seeder runs without error

**2. [Rule 3 - Blocking] EnsureCreated only creates tables for first DbContext**
- **Found during:** Task 1 (RED phase, multi-schema database setup)
- **Issue:** Multiple DbContexts with different schemas on same database; EnsureCreated is no-op after DB exists
- **Fix:** Used IRelationalDatabaseCreator.CreateTables() for subsequent contexts after first EnsureCreated
- **Files modified:** CustomWebApplicationFactory.cs
- **Verification:** All schema tables created, seeders run, tests execute

**3. [Rule 3 - Blocking] Docker Desktop not available for Testcontainers**
- **Found during:** Task 1 (RED phase)
- **Issue:** Testcontainers MsSql requires running Docker, which was not started
- **Fix:** Switched to LocalDB with unique per-test-run database names, database deleted in DisposeAsync
- **Files modified:** CustomWebApplicationFactory.cs
- **Verification:** Tests create isolated databases, run, and clean up
