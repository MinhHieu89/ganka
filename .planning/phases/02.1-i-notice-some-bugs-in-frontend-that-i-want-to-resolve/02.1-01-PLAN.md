<task type="auto">
  <name>Task 2: Add rehydration-aware bootstrap to _authenticated route guard</name>
  <files>
    frontend/src/app/routes/_authenticated.tsx
  </files>
  <action>
The `_authenticated.tsx` route guard currently checks `isAuthenticated` synchronously via `useAuthStore.getState()`. After adding persist middleware, this check happens before Zustand rehydrates from localStorage, causing false redirects to login.

**Fix the beforeLoad to handle persisted sessions:**

1. In `beforeLoad`, after checking `isAuthenticated` from `useAuthStore.getState()`:
   - If `isAuthenticated` is `true` and `refreshToken` exists, allow through (session is live in memory)
   - If `isAuthenticated` is `false`, check localStorage directly for the persisted auth key (`ganka28-auth`) as a synchronous fallback:
     - Parse the JSON from localStorage
     - If it contains a valid `refreshToken`, DON'T redirect to login yet -- the session may be valid

2. Add a `loader` function to the route that performs the actual session bootstrap:
   - Read state from `useAuthStore.getState()`
   - If `isAuthenticated` and `refreshToken` exists:
     - Attempt a silent refresh by calling the refresh API directly (use the `authClient` from `@/shared/api/client` or create a standalone fetch call to `POST /api/auth/refresh` with the stored refreshToken)
     - On success: update the store with `setAuth(user, accessToken, refreshToken)` and return
     - On failure (401, network error): call `clearAuth()` and throw `redirect({ to: '/login' })`
   - If not authenticated and no refreshToken: throw `redirect({ to: '/login' })`

3. The key insight: `beforeLoad` runs synchronously and is for quick guard checks. The `loader` is where async work (token refresh) happens. The pattern is:
   - `beforeLoad`: Check localStorage directly (synchronous) -- if no persisted auth data at all, redirect immediately
   - `loader`: If persisted data exists, attempt silent refresh before rendering

**Important:** The refresh API call in the loader should be a direct fetch (not through the React Query mutation) since this runs outside React component context. Use:
```typescript
const res = await fetch('/api/auth/refresh', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ refreshToken: state.refreshToken }),
})
```
Adjust the base URL to match the existing API client configuration (check `frontend/src/shared/api/client.ts` for the base URL pattern).

4. After successful refresh in the loader, also schedule the automatic token refresh by importing and using the appropriate mechanism, OR simply let the `useAuth` hook's existing `useEffect` handle scheduling when the component mounts with the refreshed token.
