# Phase 02.1: Frontend Bug Fixes - Context

**Gathered:** 2026-03-02
**Status:** Ready for planning

<domain>
## Phase Boundary

Fix frontend bugs discovered during Phase 02 development. Three issues: (1) session not persisting across page refresh, (2) all pages don't match shadcn/ui look and feel, and (3) refresh token stored insecurely — switch to HTTP-only cookies. Fixes and security hardening only.

</domain>

<decisions>
## Implementation Decisions

### Session Persistence
- Session is lost on every page refresh — user is always redirected to login
- Fix must ensure session persists across page refreshes (AUTH-04 compliance)
- Sessions always persist until the 30-min inactivity timeout

### Refresh Token Security (HTTP-only Cookies)
- **Switch refresh token storage from response body to HTTP-only cookie** — requires backend changes
- Backend login/refresh endpoints set refresh token via `Set-Cookie` header (HTTP-only, Secure, SameSite)
- **Clean switch** — remove refresh token from JSON response body entirely, cookie-only from now on
- **Access token: in-memory only** — stored in Zustand store memory, lost on page refresh, silently re-obtained via cookie-based refresh endpoint
- No localStorage persistence for tokens — the HTTP-only cookie handles persistence
- **Cookie scope:** `/api/auth/*` paths only — minimizes exposure, other API calls use Bearer access token
- **"Remember me" via cookie expiry:** No "Remember me" = session cookie (expires on browser close). With "Remember me" = persistent cookie (e.g., 30-day Max-Age). Backend sets appropriate expiry.

### shadcn/ui Visual Alignment
- All pages feel off from shadcn/ui design language — not specific pages
- Issues span: spacing/padding, colors/contrast, component styling, and overall layout
- **Reset to shadcn/ui default theme** — drop the custom Stone base + Green accent theme
- **Switch to rounded corners** — drop zero border radius, use shadcn/ui default border radius
- Reference https://ui.shadcn.com/create as the visual benchmark for all pages
- Every page should be audited and fixed: login, dashboard/sidebar, patient list, patient profile, scheduling/calendar, public booking

### Claude's Discretion
- Exact CSS variable values for the default shadcn/ui theme
- Which components need restyling vs which are already correct
- Order of pages to fix
- Whether to fix theme globally (CSS variables) or per-component
- Cookie configuration details (SameSite policy, Secure flag, domain scope)
- How to handle the silent token refresh flow on page load (retry logic, error states)

</decisions>
