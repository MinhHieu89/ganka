<objective>
Refactor the frontend auth flow to use HTTP-only cookies for session persistence.

Purpose: After Plan 01 switches the backend to cookie-based refresh tokens, the frontend must: (1) stop handling refresh tokens in JavaScript, (2) add `credentials: 'include'` so the browser sends cookies, and (3) implement a silent refresh flow on page load so sessions survive browser refresh. This directly fixes the session persistence bug (AUTH-04).

Output: Simplified auth flow where the refresh token is invisible to JavaScript, sessions persist across page refresh, and the access token is silently restored from the cookie-based refresh endpoint.
</objective>

  <action>
    **api-client.ts:**
    Add `credentials: "include"` to the `createClient` call so the browser sends cookies cross-origin:
    ```typescript
    export const api = createClient({ baseUrl: API_URL, credentials: "include" })
    ```
    This is required because frontend (localhost:3000) and backend (localhost:5255) are different origins. Without this, the browser will not send the refresh_token cookie. The backend CORS already has AllowCredentials().

    **auth-api.ts:**
    1. Remove `refreshToken` from `LoginResponse` interface -- the backend no longer returns it in the JSON body:
       ```typescript
       export interface LoginResponse {
         accessToken: string
         expiresAt: string
         user: UserDto
       }
       ```
    2. Remove `RefreshTokenRequest` interface entirely -- refresh no longer sends a body.
    3. Update `refreshToken` function to send no body (cookie is sent automatically by the browser):
       ```typescript
       async function refreshToken(): Promise<LoginResponse> {
         const res = await api.POST("/api/auth/refresh" as never, {})
         if (res.error) {
           const err = res.error as { detail?: string; title?: string }
           throw new Error(err.detail || err.title || "Token refresh failed")
         }
         return res.data as LoginResponse
       }
       ```
    4. Update `useRefreshMutation` to match new no-argument signature:
       ```typescript
       export function useRefreshMutation() {
         return useMutation({
           mutationFn: refreshToken,  // no args needed
         })
       }
       ```
    5. Add a standalone `silentRefresh` function (not a hook) for use in `beforeLoad`:
       ```typescript
       export async function silentRefresh(): Promise<LoginResponse | null> {
         try {
           const res = await api.POST("/api/auth/refresh" as never, {})
           if (res.error) return null
           return res.data as LoginResponse
         } catch {
           return null
         }
       }
       ```
       This function is separate from the mutation because `beforeLoad` runs outside React component context (no hooks allowed).

    **_authenticated.tsx:**
    Make `beforeLoad` async with silent refresh attempt:
    ```typescript
    import { createFileRoute, redirect } from "@tanstack/react-router"
    import { AppShell } from "@/shared/components/AppShell"
    import { useAuthStore } from "@/shared/stores/authStore"
    import { silentRefresh } from "@/features/auth/api/auth-api"

    export const Route = createFileRoute("/_authenticated")({
      beforeLoad: async ({ location }) => {
        const { isAuthenticated, accessToken } = useAuthStore.getState()

        // If already authenticated with access token in memory, proceed
        if (isAuthenticated && accessToken) {
          return
        }

        // Attempt silent refresh using HTTP-only cookie
        try {
          const response = await silentRefresh()
          if (response) {
            useAuthStore.getState().setAuth(
              {
                id: response.user.id,
                email: response.user.email,
                fullName: response.user.fullName,
                permissions: response.user.permissions,
                preferredLanguage: response.user.preferredLanguage,
              },
              response.accessToken,
            )
            return // session silently restored
          }
        } catch {
          // Silent refresh failed -- fall through to login redirect
        }

        // No valid session -- redirect to login
        throw redirect({
          to: "/login",
          search: { redirect: location.href },
        })
      },
      component: AppShell,
    })
    ```
