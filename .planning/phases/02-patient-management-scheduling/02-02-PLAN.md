Build the complete Scheduling module backend as a vertical slice: appointment domain with double-booking prevention, self-booking workflow with staff approval, clinic operating hours enforcement, and both authenticated and public API endpoints.

Purpose: Provides the backend API for appointment booking (staff + patient self-service), calendar data retrieval, and appointment lifecycle management. Double-booking prevention uses both application-level overlap checks and database-level unique filtered index as safety net.
Output: Fully functional Scheduling module with domain, contracts, application, infrastructure, and presentation layers (including unauthenticated public booking endpoints) registered in the Bootstrapper.

       - Load appointment and its type for duration
       - Compute new EndTime
       - Validate clinic schedule for new time
       - Check overlap (exclude current appointment via excludeAppointmentId)
       - Call appointment.Reschedule(newStart, newEnd)
       - Save. Return Result.

       **GetAppointmentsByDoctor.cs:** Query(DoctorId, DateFrom, DateTo) + Handler:
       - Call GetByDoctorAsync, map to List<AppointmentDto>

       - Validate clinic schedule and overlap (same as BookAppointment)
       - Create Appointment from the request data
       - Call request.Approve(appointment.Id)
       - Save both. Return Result<Guid> (appointment Id)
