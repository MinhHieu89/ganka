  <action>
    **Rewrite the allergy name field in AllergyForm.tsx** to fix all three issues: Input-as-PopoverTrigger anti-pattern, missing free-text confirmation, and hardcoded English categories.

    **Architecture of the new autocomplete:**
    - Input sits OUTSIDE PopoverTrigger (always editable, no toggle behavior)
    - Popover opens on input focus, closes on blur with a small delay (150ms) to allow item clicks
    - Command component uses `shouldFilter={false}` to disable cmdk's internal filtering (we handle filtering ourselves)
    - Filtered catalog items shown in CommandGroup with localized category labels
    - A dynamic "Add custom: {inputValue}" CommandItem appears when the typed text does not exactly match any catalog item -- this is the free-text confirmation affordance
    - When selecting a catalog item: store `item.en` as form value, show localized label in input
    - When selecting the "Add custom" item: store the raw typed text as form value

    **Implementation details:**

    1. Add a category i18n key mapping in the component:
    ```tsx
    const categoryKeyMap: Record<string, string> = {
      "Ophthalmic Drug": "allergyCategory.ophthalmicDrug",
      "General Drug": "allergyCategory.generalDrug",
      "Material": "allergyCategory.material",
      "Environmental": "allergyCategory.environmental",
    }
    ```

    2. In the catalogItems mapping, add the translated category:
    ```tsx
    const catalogItems = ALLERGY_CATALOG_BILINGUAL.map((item) => ({
      label: i18n.language === "vi" ? item.vi : item.en,
      value: item.en,
      category: item.category,
      categoryLabel: t(categoryKeyMap[item.category] ?? item.category),
    }))
    ```

    3. Replace the entire Popover+Command block with the corrected pattern:
    ```tsx
    <Field data-invalid={fieldState.invalid || undefined}>
      <FieldLabel>{t("allergyName")}</FieldLabel>
      <Popover open={popoverOpen} onOpenChange={setPopoverOpen}>
        <PopoverTrigger asChild>
          <div>
            <Input
              value={inputValue}
              placeholder={t("allergyName")}
              aria-invalid={fieldState.invalid || undefined}
              onFocus={() => setPopoverOpen(true)}
              onChange={(e) => {
                const val = e.target.value
                setInputValue(val)
                field.onChange(val)
                if (!popoverOpen) setPopoverOpen(true)
              }}
              autoComplete="off"
            />
          </div>
        </PopoverTrigger>
        <PopoverContent
          className="w-[--radix-popover-trigger-width] p-0"
          align="start"
          onOpenAutoFocus={(e) => e.preventDefault()}
        >
          <Command shouldFilter={false}>
            <CommandList>
              {filtered.length === 0 && !inputValue.trim() && (
                <CommandEmpty>{tCommon("search.noResults")}</CommandEmpty>
              )}
              {/* Show "Add custom" option when typed text doesn't match any item exactly */}
              {inputValue.trim() && !catalogItems.some(
                (item) => item.label.toLowerCase() === inputValue.toLowerCase() ||
                          item.value.toLowerCase() === inputValue.toLowerCase()
              ) && (
                <CommandItem
                  value={`custom:${inputValue}`}
                  onSelect={() => {
                    field.onChange(inputValue.trim())
                    setPopoverOpen(false)
                  }}
                >
                  <IconPlus className="mr-2 h-4 w-4" />
                  {t("allergyCategory.custom")}: "{inputValue.trim()}"
                </CommandItem>
              )}
              {/* Group filtered catalog items by category */}
              {Object.entries(
                filtered.reduce<Record<string, typeof filtered>>((acc, item) => {
                  const cat = item.categoryLabel
                  ;(acc[cat] ??= []).push(item)
                  return acc
                }, {})
              ).map(([categoryLabel, items]) => (
                <CommandGroup key={categoryLabel} heading={categoryLabel}>
                  {items.map((item) => (
                    <CommandItem
                      key={item.value}
                      value={item.value}
                      onSelect={() => {
                        field.onChange(item.value)
                        setInputValue(item.label)
                        setPopoverOpen(false)
                      }}
                    >
                      <IconCheck
                        className={cn(
                          "mr-2 h-4 w-4",
                          nameValue === item.value ? "opacity-100" : "opacity-0"
                        )}
                      />
                      {item.label}
                    </CommandItem>
                  ))}
                </CommandGroup>
              ))}
            </CommandList>
          </Command>
        </PopoverContent>
      </Popover>
      {fieldState.error && (
        <FieldError>{fieldState.error.message}</FieldError>
      )}
    </Field>
    ```

    Key differences from current implementation:
    - `shouldFilter={false}` on Command prevents cmdk internal filtering conflict
    - Input wrapped in a plain `<div>` inside PopoverTrigger to avoid the click-to-toggle behavior on the input itself (the div is the trigger but the input handles its own focus/change)
    - Dynamic "Add custom" CommandItem with IconPlus icon appears when no exact match
    - Category items grouped by translated category heading
    - Selecting catalog item sets `item.value` (English) as form value and `item.label` (localized) as display text

    4. Also add the `IconPlus` import from `@tabler/icons-react`.
