    1. **OpenShift.cs**:
       - OpenShiftCommand: sealed record(decimal OpeningBalance, Guid? ShiftTemplateId)
       - Validator: OpeningBalance >= 0
       - Handler: Check no existing open shift via ICashierShiftRepository.GetCurrentOpenAsync. Get cashier info from ICurrentUser (userId, name). Create CashierShift.Create(...). Save. Return Result<CashierShiftDto>.

    2. **CloseShift.cs**:
       - CloseShiftCommand: sealed record(decimal ActualCashCount, string? ManagerNote)
       - Validator: ActualCashCount >= 0
       - Handler: Get current open shift. Call shift.LockForClose(). Then shift.Close(actualCashCount, managerNote). Save. Return Result<CashierShiftDto>.
       - Note: In production, Lock and Close could be separate endpoints for the race condition prevention described in research. For simplicity, do both in one handler -- Lock prevents new payment assignments, then Close calculates discrepancy.

    3. **GetCurrentShift.cs**:
       - GetCurrentShiftQuery: sealed record() (no params, uses ICurrentUser for BranchId)
       - Handler: Get current open shift from repo, map to CashierShiftDto or return null Result.

    4. **GetShiftReport.cs**:
       - GetShiftReportQuery: sealed record(Guid ShiftId)
       - Handler: Load shift by ID. Load all payments for the shift via IPaymentRepository.GetByShiftIdAsync. Group payments by Method, sum amounts per method to build revenue-by-method dictionary. Map to ShiftReportDto with all shift fields + computed revenue breakdown.
