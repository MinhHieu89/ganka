  <action>
    Create Services/ directory in Billing.Infrastructure.

    1. **BillingDocumentService.cs**: Implements IBillingDocumentService:
       - Constructor injects IInvoiceRepository, ICashierShiftRepository, IClinicSettingsService (from Shared.Application), IPaymentRepository
       - GenerateInvoicePdfAsync: Load invoice with all children, load clinic settings, build InvoiceDocument (from Plan 15), call GeneratePdf(), return byte[]
       - GenerateReceiptPdfAsync: Similar but uses ReceiptDocument (from Plan 15)
       - GenerateEInvoicePdfAsync: Load invoice, map to EInvoiceExportDto (add tax calculation), build EInvoiceDocument (from Plan 15), call GeneratePdf()
       - GenerateShiftReportPdfAsync: Load shift, load payments for shift, build ShiftReportDocument (from Plan 15), call GeneratePdf()
       - Map domain entities to document data classes
       - Follow the pattern from Clinical.Infrastructure DocumentService (Phase 5)

    2. **EInvoiceExportService.cs**: Separate service for structured data export:
       - ExportJsonAsync(Guid invoiceId, CancellationToken ct) -> string: Load invoice, map to EInvoiceExportDto with all required fields, serialize to JSON using System.Text.Json with Vietnamese-friendly options (no escaping Unicode)
       - ExportXmlAsync(Guid invoiceId, CancellationToken ct) -> string: Same data, serialize to XML using System.Xml.Linq (XDocument/XElement pattern)
       - Tax calculation: SubTotal * TaxRate (default 8%) = TaxAmount. TotalWithTax = SubTotal - DiscountTotal + TaxAmount
       - Seller info from IClinicSettingsService (name, tax code, address)
       - Buyer info from invoice (patient name, optional tax code)
       - Both formats suitable for MISA import
  </action>
