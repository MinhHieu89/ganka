Key insight: Entities loaded via `GetByIdAsync()` are already tracked by EF Core's change tracker. Calling `context.Invoices.Update(invoice)` marks ALL properties as Modified, including the `RowVersion` column which is a SQL Server `rowversion` (auto-generated, read-only). EF then tries to include RowVersion in the UPDATE SET clause, which SQL Server rejects. The fix is to simply remove the Update() call -- EF Core's change tracker automatically detects modifications to tracked entities on SaveChangesAsync().

The `AddLineItemHandler` and `FinalizeInvoiceHandler` already follow the correct pattern (no Update() call). The handlers below incorrectly call Update().
