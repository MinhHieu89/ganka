    1. **ShiftTemplate.cs**: Entity with:
       - string Name (e.g., "Morning", "Afternoon")
       - string NameVi (e.g., "Ca sang", "Ca chieu")
       - TimeOnly DefaultStartTime
       - TimeOnly DefaultEndTime
       - bool IsActive (soft delete)
       - Private constructor, static Create factory

    2. **CashierShift.cs**: AggregateRoot implementing IAuditable with:
       - Guid CashierId (user who opened the shift)
       - string CashierName (denormalized)
       - Guid? ShiftTemplateId (nullable -- custom shift if no template)
       - ShiftStatus Status (default Open)
       - DateTime OpenedAt
       - DateTime? ClosedAt
       - decimal OpeningBalance (cash in drawer at start)
       - decimal ExpectedCashAmount (computed: OpeningBalance + CashReceived - CashRefunds)
       - decimal CashReceived (sum of confirmed cash payments during shift)
       - decimal CashRefunds (sum of cash refunds during shift)
       - decimal? ActualCashCount (physical count entered at close)
       - decimal? Discrepancy (ActualCashCount - ExpectedCashAmount)
       - string? ManagerNote (explanation for discrepancy)
       - decimal TotalRevenue (sum of all confirmed payments across all methods)
       - int TransactionCount (number of finalized invoices during shift)
       - Private constructor
       - Static Create(Guid cashierId, string cashierName, decimal openingBalance, Guid? shiftTemplateId, BranchId branchId)
       - LockForClose() -- sets Status to Locked, prevents new payment assignments
       - Close(decimal actualCashCount, string? managerNote) -- validates Status is Locked, sets ActualCashCount, computes Discrepancy, sets Status to Closed, ClosedAt
       - AddCashReceived(decimal amount) -- increments CashReceived and TotalRevenue
       - AddNonCashRevenue(decimal amount) -- increments TotalRevenue only
       - AddCashRefund(decimal amount) -- increments CashRefunds
       - IncrementTransactionCount()
       - EnsureOpen() -- validates shift is Open status
       - EnsureLocked() -- validates shift is Locked for close

    Business rules:
    - Only one open shift per branch (enforced at application layer, not domain)
    - Cash reconciliation: Discrepancy = ActualCashCount - ExpectedCashAmount
    - ExpectedCashAmount = OpeningBalance + CashReceived - CashRefunds
