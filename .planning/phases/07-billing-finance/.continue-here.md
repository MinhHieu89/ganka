Running full UAT re-test of Phase 07 (Billing & Finance) using Playwright + API calls.
Tests 1-31 completed via API, tests 32-43 (shift close, shift history, PDFs, UI details) not yet run.

A critical root cause bug was discovered and FIXED during this UAT session:
- **EF Core `ValueGeneratedOnAdd` vs client-generated Guid IDs** — all entities generate IDs in the constructor (`Id = Guid.NewGuid()`), but EF Core's default `ValueGeneratedOnAdd` treated new child entities added to navigation collections as `Modified` (UPDATE) instead of `Added` (INSERT). This caused `DbUpdateConcurrencyException` on Invoice because it has a RowVersion concurrency token. The UPDATE on the child affected 0 rows silently, then the Invoice UPDATE with WHERE RowVersion=@old_value also failed.
- **Fix applied**: Added `ValueGenerated = Never` loop in `OnModelCreating` of ALL 9 module DbContexts.
