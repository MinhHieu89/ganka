    AddInvoiceLineItem:
    - Given valid InvoiceId in Draft status, description, unitPrice, quantity, department -> adds line item and recalculates totals
    - Given Finalized invoice -> returns error "Invoice is not in draft status"
    - Department field is set for revenue allocation (FIN-02)

    FinalizeInvoice:
    - Given fully paid invoice in Draft status and valid CashierShiftId -> sets status to Finalized, records finalizer and timestamp
    - Given invoice with outstanding balance -> returns error "Cannot finalize invoice with outstanding balance"
    - Given already finalized invoice -> returns error
  </behavior>

    4. Implement AddInvoiceLineItem.cs: Command with InvoiceId + line item fields, Handler loads invoice, calls AddLineItem domain method, saves
    5. Implement FinalizeInvoice.cs: Command with InvoiceId + CashierShiftId, Handler loads invoice, calls Finalize domain method, saves
