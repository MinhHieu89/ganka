To make the invoice detail page auto-refresh when line items are added:

1. In `InvoiceView.tsx`, import and call the `useBillingHub` hook:
   ```tsx
   import { useBillingHub } from "@/features/billing/hooks/use-billing-hub"
   ```
   Add inside the `InvoiceView` component function, near the top with other hooks:
   ```tsx
   useBillingHub() // Real-time updates when prescription items are added
   ```

That is all that is needed. The existing "LineItemAdded" handler in use-billing-hub.ts already invalidates `billingKeys.invoice(notification.invoiceId)`, which will cause `useInvoice(invoiceId)` to refetch, and React will re-render InvoiceView with the new line items.

Important: Do NOT add `refetchInterval` polling -- the SignalR approach is already implemented and is the correct real-time mechanism. The hook handles connection, reconnection, and group membership automatically.

Note: The backend HandleDrugPrescriptionAdded handler already calls `notificationService.NotifyLineItemAddedAsync()` which sends the "LineItemAdded" event via SignalR. So the full chain is: doctor prescribes -> domain event -> integration event -> billing handler creates line items + sends SignalR -> frontend hook invalidates query -> InvoiceView re-renders.
