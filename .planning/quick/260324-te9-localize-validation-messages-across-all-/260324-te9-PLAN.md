For EACH form file with hardcoded validation messages, apply this transformation pattern:

**Pattern A -- Forms with static schema defined outside the component (most common):**

BEFORE:
```typescript
const frameFormSchema = z.object({
  brand: z.string().min(1, "required").max(100),
  sellingPrice: z.number().positive("Must be greater than 0"),
  lensWidth: z.number().int().min(40, "Must be between 40 and 65").max(65, "Must be between 40 and 65"),
})
// ... inside component:
const form = useForm({ resolver: zodResolver(frameFormSchema) })
```

AFTER:
```typescript
import { createValidationMessages } from "@/shared/lib/validation"

function createFrameFormSchema(t: (key: string, opts?: Record<string, unknown>) => string) {
  const v = createValidationMessages(t)
  return z.object({
    brand: z.string().min(1, v.required).max(100),
    sellingPrice: z.number().positive(v.mustBePositive),
    lensWidth: z.number().int().min(40, v.between(40, 65)).max(65, v.between(40, 65)),
  })
}
// ... inside component:
const { t: tCommon } = useTranslation("common")
const schema = useMemo(() => createFrameFormSchema(tCommon), [tCommon])
const form = useForm({ resolver: zodResolver(schema) })
```

**Pattern B -- Forms already using tCommon/t but with some hardcoded strings:**
Just replace the hardcoded strings with the appropriate `v.*` call. These forms already have the `t` function available.

**Pattern C -- Forms using namespace-specific `t` but for generic messages (e.g., DrugPrescriptionForm):**
```typescript
// BEFORE: t("prescription.drugName") + " required"
// AFTER: use tCommon for generic validation, keep t for domain labels
drugName: z.string().min(1, v.required),
```

**Specific mappings for hardcoded strings found:**

| Hardcoded string | Replacement |

| `"Must be greater than 0"` / `"must be > 0"` | `v.mustBePositive` |
| `"Must be >= 0"` / `"must be >= 0"` | `v.mustBeNonNegative` |

| `"At least one line item required"` | `v.minItems(1)` |
